# TagInput

Multi-tag entry. Tags appear inside the input as removable Badges. Enter, Tab, and any `commitOn` character (default comma) commits the current text. Backspace on an empty input removes the trailing tag.

Use for filters, labels, recipients, "tags" or "categories" pickers where the values are open-ended. For closed sets, prefer [Combobox](/docs/components/combobox) or a multi-select [Menu](/docs/components/menu).

## Usage

```tsx
import { TagInput } from "@patchui/react";

const [tags, setTags] = useState<string[]>([]);

<TagInput
  value={tags}
  onValueChange={setTags}
  placeholder="Add a tag — Enter or comma"
/>

// Custom commit characters (commit on Enter, comma, semicolon)
<TagInput commitOn={[",", ";"]} />

// Max tags
<TagInput max={5} />

// Custom validation
<TagInput
  validate={(candidate, current) =>
    candidate.length >= 2 || "Tag too short"
  }
/>

// Variants match Input
<TagInput variant="outlined" />  // default
<TagInput variant="ghost" />
<TagInput variant="underline" />

// Error state
<TagInput invalid />
```

<Callout title="Notes">
- **Keyboard:** `Enter` / `Tab` / any `commitOn` character commits the current draft. `Backspace` on empty input removes the trailing tag. The native input is fully editable for typing — there's no need for arrow-key navigation since tag removal is via the per-tag × button.
- **Dedupe:** When `dedupe={true}` (default), submitting a tag already in the list silently clears the draft instead of adding a duplicate. The check is case-insensitive.
- **Transform + validate:** Transform runs first (default: trim), then validate decides if the candidate is committable. Combine them for normalized inputs (e.g. `transform={(raw) => raw.toLowerCase()}` for tag normalization).
- **Composition with Combobox:** TagInput is for free-text entry. If you need autocomplete suggestions, compose Combobox + the same chip pattern manually — keep this primitive focused.
</Callout>
