# NavigationMenu

A horizontal navigation bar built on Base UI's NavigationMenu. Items are either a plain link or a trigger with a dropdown panel; multiple panels share one popup that morphs (size and position) between them. It is designed to drop straight into a header, including inside `AppHeader`. The popup scaffold (portal, positioner, popup, viewport) is rendered for you, so you only write the list.

## Usage

```tsx
import {
  NavigationMenu,
  NavigationMenuList,
  NavigationMenuItem,
  NavigationMenuTrigger,
  NavigationMenuContent,
  NavigationMenuLink,
} from "@patchui/react";

<NavigationMenu>
  <NavigationMenuList>
    <NavigationMenuItem>
      <NavigationMenuTrigger>Product</NavigationMenuTrigger>
      <NavigationMenuContent>
        <div className="grid w-[440px] grid-cols-2 gap-1">
          <NavigationMenuLink href="/features">Features</NavigationMenuLink>
          <NavigationMenuLink href="/pricing">Pricing</NavigationMenuLink>
        </div>
      </NavigationMenuContent>
    </NavigationMenuItem>
    <NavigationMenuItem>
      <NavigationMenuLink href="/docs">Docs</NavigationMenuLink>
    </NavigationMenuItem>
  </NavigationMenuList>
</NavigationMenu>
```

Use the `render` prop on `NavigationMenuLink` to integrate your router (e.g. a Next.js `<Link>`): `<NavigationMenuLink render={<Link href="/docs" />}>Docs</NavigationMenuLink>`.

## Components

- `NavigationMenu` - root; also renders the shared popup (portal, positioner, popup, viewport) automatically
- `NavigationMenuList` - the horizontal list of items
- `NavigationMenuItem` - a single item (a link, or a trigger plus content)
- `NavigationMenuTrigger` - opens a dropdown panel; includes a chevron that rotates on open
- `NavigationMenuContent` - the panel for an item; rendered into the shared morphing popup
- `NavigationMenuLink` - a navigable link, in the bar or inside content

<Callout title="Notes">
- **Fits AppHeader:** drop `NavigationMenu` between `AppHeaderBrand` and `AppHeaderRight` for a product-style header.
- **Morphing popup:** all `NavigationMenuContent` panels share one radius-12 popup with a 1px hairline boundary and `--shadow-menu` elevation. It animates size and position between items with a direction-aware slide.
- **Hover tokens:** Triggers and links use `--fill-1` for hover and open states, matching Menu item hover styling.
- **Keyboard + a11y:** Base UI handles roving focus, arrow-key navigation, and the ARIA wiring. Triggers use the compact non-editable focus indicator.
</Callout>
