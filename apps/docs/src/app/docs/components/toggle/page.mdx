# Toggle

A press-to-toggle button with persistent on / off state. Use for actions with state: star ★, pin 📌, mute, bookmark, follow, formatting toolbars (bold / italic / underline).

Distinct from `Switch` (binary form-state control) and `Button` (one-shot action). Toggle has `role="button"` + `aria-pressed` and a visual "on" state that persists until toggled off.

## Usage

```tsx
import { Toggle } from "@patchui/react";
import { Star } from "lucide-react";

// Controlled
const [starred, setStarred] = useState(false);
<Toggle pressed={starred} onPressedChange={setStarred} icon={<Star />} aria-label="Star" />

// Uncontrolled with initial state
<Toggle defaultPressed icon={<Star />} aria-label="Star" />

// With text
<Toggle>Public</Toggle>

// Outline variant
<Toggle variant="outline" icon={<Bold />} aria-label="Bold" />

// Sizes
<Toggle size="sm" />
<Toggle size="md" />  // default
<Toggle size="lg" />
```

<Callout title="Notes">
- **Pressed visual:** When `pressed={true}`, background switches to `--patch-text` and text to `--patch-bg` — high-contrast inverted, instantly readable as "on".
- **`aria-pressed`:** Screen readers announce "Star, toggle button, pressed" / "not pressed". Provide `aria-label` for icon-only toggles.
- **vs Switch:** Switch is a form control representing a setting (write to disk / save to state). Toggle is an action with state (star this item / pin to top). Switch reads as a noun ("Dark mode: on"); Toggle reads as a verb ("Star").
- **vs SegmentedToggle:** Toggle is single-button with on/off state. SegmentedToggle is a multi-option radio group ("Grid / List view").
</Callout>
