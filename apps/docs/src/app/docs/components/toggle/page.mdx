# Toggle

A press-to-toggle button with persistent on / off state. Use for actions with state: star ★, pin 📌, mute, bookmark, follow, formatting toolbars (bold / italic / underline).

Distinct from `Switch` (binary form-state control) and `Button` (one-shot action). Toggle has `role="button"` + `aria-pressed` and a visual "on" state that persists until toggled off.

## Usage

```tsx
import { Toggle } from "@patchui/react";
import { Star } from "lucide-react";

// Controlled
const [starred, setStarred] = useState(false);
<Toggle pressed={starred} onPressedChange={setStarred} icon={<Star />} aria-label="Star" />

// Uncontrolled with initial state
<Toggle defaultPressed icon={<Star />} aria-label="Star" />

// With text
<Toggle>Public</Toggle>

// Outline variant
<Toggle variant="outline" icon={<Bold />} aria-label="Bold" />

// Sizes
<Toggle size="sm" />
<Toggle size="md" />  // default
<Toggle size="lg" />
```

<Callout title="Notes">
- The pressed state flips the fill to `gray-1000` with `background-100` text — inverted so the pressed button reads as "on" at a glance.
- Rendered as a `<button>` with `aria-pressed`, so screen readers announce "Star, toggle button, pressed" / "not pressed". Provide `aria-label` for icon-only toggles.
- For a binary on/off setting slider (Dark mode, Enable notifications), reach for `Switch` instead — Switch reads as a noun (a setting), Toggle reads as a verb (an action).
- Reach for `Toggle` when the state is about the toolbar action (`Star` this item, `Pin` to top). Reach for `Switch` when the state is a setting (`Dark mode`, `Enable Firewall`). Toggle reads as a verb, Switch as a noun.
- `SegmentedToggle` is the multi-choice sibling — one active option out of several (Grid / List view).
</Callout>
