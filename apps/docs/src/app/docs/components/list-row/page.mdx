# ListRow

A two-column row for stacked lists of heterogeneous items — member rows, integration rows, notification rows, watchlist items. Each row has an optional `left` slot (avatar / checkbox / icon), a body region (title + description or arbitrary content), and an optional `right` slot (button / actions).

Reach for `ListRow` when the rows share a shape but don't have sortable columns. If your data IS tabular (compare across columns), use `Table` instead.

## Usage

```tsx
import {
  Avatar,
  Button,
  ListRow,
  ListRowContent,
  ListRowList,
} from "@patchui/react";

<ListRowList as="ul">
  <ListRow
    as="li"
    left={<Avatar letter="A" />}
    right={<Button size="sm" variant="tertiary">Manage</Button>}
  >
    <ListRowContent title="Ada Lovelace" description="ada@example.com" />
  </ListRow>
</ListRowList>
```

<Callout title="Notes">
- **ListRow vs Table.** Reach for ListRow when rows share a shape but not a column model (member rows, integration rows, notification rows). Reach for Table when data is genuinely tabular (multiple sortable / comparable columns).
- **Right-cluster restraint.** Vercel guidance: cap the right slot at one or two controls. When there's more, put secondary actions behind a Menu.
- **Left slot leads with a scannable identifier.** Avatar or icon first, then title, then metadata — the eye lands on the identifier and moves rightward.
- **Loading state.** Compose `<Skeleton>` inside the left / body / right slots to preserve layout while data loads. `<Skeleton show={loading}>...</Skeleton>` wraps the real content and flips once loaded.
- **Interactive rows.** Set `interactive` on the list to add a subtle hover wash. Wrap the row in an `<a>` or `<button>` for keyboard focus + click semantics.
</Callout>
