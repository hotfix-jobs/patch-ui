# SegmentedToggle

A compact segmented radio control. Use for view-mode toggles (grid / list), sort direction, density, or any small set of mutually-exclusive options. The active background uses motion's `layoutId` so it **physically slides** between buttons when the selection changes.

## Usage

```tsx
import {
  SegmentedToggle,
  SegmentedToggleItem,
} from "@patchui/react";
import { LayoutGrid, Rows3 } from "lucide-react";

// Icon-only view toggle
<SegmentedToggle value={view} onValueChange={setView} aria-label="View">
  <SegmentedToggleItem value="grid" aria-label="Grid view">
    <LayoutGrid />
  </SegmentedToggleItem>
  <SegmentedToggleItem value="list" aria-label="List view">
    <Rows3 />
  </SegmentedToggleItem>
</SegmentedToggle>

// Text labels
<SegmentedToggle value={sort} onValueChange={setSort} aria-label="Sort">
  <SegmentedToggleItem value="newest">Newest</SegmentedToggleItem>
  <SegmentedToggleItem value="oldest">Oldest</SegmentedToggleItem>
</SegmentedToggle>

// Small size for tight chrome
<SegmentedToggle size="sm" value={view} onValueChange={setView} aria-label="View">
  ...
</SegmentedToggle>
```

<Callout title="Notes">
- Reach for SegmentedToggle when 2–3 options are mutually exclusive and different views of the same surface — `Grid` / `List`, `Source` / `Output`, alignment picker. Past 3 options, use `Tabs` or a `Select`.
- The active pill is a shared `motion.div` with a `layoutId` per group, so switching selections physically slides the pill between items. Reduced motion swaps the pill instantly.
- Icon-only items need their own `aria-label` — the icon alone doesn't announce the choice.
- Container is `role="radiogroup"` with `aria-label`; items are `role="radio"` with `aria-checked`. Only the selected item is in the Tab order. Arrows / Home / End move within the group.
- Multiple SegmentedToggles on the same page each get a unique `layoutId` namespace via `useId`, so pills don't cross-animate.
</Callout>
