# Checkbox

Two-state control (checked / unchecked) with an indeterminate visual for partial-selection headers. Pass a label as children and the whole label becomes the click target automatically.

<Install name="checkbox" />

<ComponentPreview label="Checkbox Examples">

## Usage

```tsx
import { Checkbox } from "@patchui/react";

// With label as children — the label becomes clickable
<Checkbox>Send weekly digest</Checkbox>

// Controlled
const [checked, setChecked] = useState(false);
<Checkbox checked={checked} onCheckedChange={setChecked}>
  Notify on mentions
</Checkbox>

// Uncontrolled with initial state
<Checkbox defaultChecked>Enabled by default</Checkbox>

// Indeterminate (drive from a parent that knows partial selection)
<Checkbox indeterminate>3 of 5 selected</Checkbox>

// Bare (no label) — for row-select in tables, use aria-label
<Checkbox aria-label="Select row" />

// Acknowledgment (full sentence, ends in period)
<Checkbox required>I agree to the Terms of Service.</Checkbox>

// Disabled
<Checkbox disabled>Locked option</Checkbox>
```

<Callout title="Notes">
- Reach for Checkbox when the user is picking zero or more from a list (opt-in preferences, table row select, filter multi-select) or affirming a specific statement (terms, exports). Reach for `Switch` when a single boolean setting saves immediately.
- Indeterminate is only visual. Drive it from a parent that tracks partial selection: as soon as every child is either fully on or fully off, clear the indeterminate state.
- For row-select checkboxes with no visible label, pass `aria-label="Select {row name}"` so the row stays identifiable when the surrounding context is off-screen.
- Group related checkboxes in a `<fieldset>` with a `<legend>` so screen readers announce the group name before each option.
- The checkbox is a `<button role="checkbox">` on top of a hidden `<input type="checkbox">`, so it participates in native forms without losing keyboard behavior.
</Callout>
