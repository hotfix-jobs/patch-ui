# Checkbox

A single boolean choice for forms, with controlled, uncontrolled, indeterminate, and disabled states.

<Install name="checkbox" />

<ComponentPreview label="Checkbox Examples">
  <CheckboxDemo />
</ComponentPreview>

## Usage

```tsx
import { Checkbox } from "@patchui/react";

// Uncontrolled
<Checkbox defaultChecked />

// Controlled
const [checked, setChecked] = useState(false);
<Checkbox checked={checked} onCheckedChange={(c) => setChecked(c)} />

// Indeterminate (mixed state in a parent / child checkbox group)
<Checkbox indeterminate />

// Disabled
<Checkbox disabled />
```

<Callout title="Notes">
- **Checkbox vs Switch:** Use a checkbox for form options that take effect on submit (e.g. "I agree to the terms"). Use a [Switch](/docs/components/switch) for settings that apply immediately.
- **Indeterminate:** A visual-only state, useful for "select all" headers when only some children are ticked. The native input is still reported as unchecked.
- **Accessibility:** Renders a hidden `<input type="checkbox">` for native form participation. Wrap the checkbox and its label in a `<label>` so clicking the label toggles the checkbox.
- **Controlled vs uncontrolled:** Use `checked` + `onCheckedChange` for controlled usage. Use `defaultChecked` for uncontrolled.
</Callout>
