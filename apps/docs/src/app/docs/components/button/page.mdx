# Button

A polymorphic button component with five variants, three sizes, icon support, four corner shapes, and a loading state.

  <ButtonDemo />
</ComponentPreview>

## Usage

```tsx
import { Button } from "@patchui/react";

// Basic
<Button variant="primary">Deploy Project</Button>

// Variants
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="tertiary">Tertiary</Button>
<Button variant="warning">Warning</Button>
<Button variant="destructive">Destructive</Button>

// Sizes: 24 / 32 / 40 px
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>

// Icon-only + shape (aria-label required)
<Button shape="square" aria-label="Add" icon={<Plus />} />
<Button shape="circle" aria-label="Add" icon={<Plus />} />

// Marketing pill shape (typically paired with shadow)
<Button shape="pill" shadow variant="primary" icon={<ArrowRight />} iconPosition="right">
  Start Free Trial
</Button>

// Muted (chip / filter chrome). Keeps the variant's surface, mutes the label + icon.
<Button variant="secondary" muted icon={<Globe />}>Visit site</Button>

// Dismissible filter chip (click focuses, × removes)
<Button
  size="sm"
  shape="pill"
  variant="secondary"
  icon={<Hash />}
  onClick={focusFilter}
  onRemove={removeFilter}
>
  design
</Button>

// Loading state
<Button loading>Saving…</Button>

// As a link (polymorphic render prop)
<Button render={<a href="/dashboard" />}>Go to Dashboard</Button>
```

<Callout title="Notes">
- Each variant maps to a specific token: `primary` reads the `--primary` / `--on-primary` chain (monochrome by default, brand-overridable), `secondary` = `surface-1` + `hairline` border (elevates over the page), `tertiary` = transparent + hover tint, `warning` = fixed `--warning` role, `destructive` = fixed `--error` role.
- Reserve `primary` for the one hero action on a view. Reserve `destructive` for actions the user cannot undo without work.
- Brand override: redefine `--primary`, `--on-primary`, `--primary-hover`, and `--primary-active` in a consumer stylesheet after the token import to swap the monochrome default for a chromatic brand color. Every primary Button picks it up without callsite edits.
- Icon-only buttons: pass `icon` with no `children`. The button squares up to match the size. `aria-label` is required; a dev-time warning fires when it's missing.
- `shape="pill"` alongside `shadow` produces the marketing hero CTA: full-radius pill with a soft elevation shadow. Reserve for hero and closing-CTA rows on landing pages, not general controls.
- Polymorphism: pass `render={<a href="…" />}` (or a `<Link>`) to render as an anchor with the button's styling and behavior.
- `loading` swaps the content for a spinner and disables interaction so a double-tap can't fire twice.
- Every size ships a hidden 44×44 touch target underneath, so the tap area works on coarse pointers even for a small button.
- Labels: Title Case, verb + noun (`Deploy Project`, `Delete Member`). A lone `Submit` or `OK` tells the user nothing about what will happen.
</Callout>
