# Button

A polymorphic button with intent-based variants, three sizes, rounded or pill shapes, icon support, and a loading state. Icon-only controls keep the selected shape.

## Usage

```tsx
import { Button } from "@patchui/react";

// Basic
<Button variant="primary">Deploy Project</Button>

// Variants
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="tertiary">Tertiary</Button>
<Button variant="warning">Warning</Button>
<Button variant="destructive">Destructive</Button>

// Sizes: 24 / 32 / 40 px
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>

// Icon-only Buttons keep the default rounded shape (aria-label required)
<Button aria-label="Add" icon={<Plus />} />

// Opt into a circular icon button
<Button aria-label="Add" shape="pill" icon={<Plus />} />

// Pill is opt-in for friendly CTAs and filter actions
<Button shape="pill">Browse Items</Button>

// Loading state
<Button loading>Saving…</Button>

// As a link (polymorphic render prop)
<Button render={<a href="/dashboard" />}>Go to Dashboard</Button>
```

<Callout title="Notes">
- Each variant maps to a specific token: `primary` reads the `--primary` / `--on-primary` chain, `secondary` uses the neutral fill ladder, `soft` uses `--primary-soft`, `tertiary` is transparent with a hover tint, and semantic actions use their fixed role colors.
- Reserve `primary` for the one hero action on a view. Reserve `destructive` for actions the user cannot undo without work.
- Brand override: redefine `--primary`, `--on-primary`, `--primary-hover`, and `--primary-active` in a consumer stylesheet after the token import to swap the monochrome default for a chromatic brand color. Every primary Button picks it up without callsite edits.
- Icon-only buttons: pass `icon` with no `children`. Button infers equal width and height while respecting `shape`; use `shape="pill"` when a circle is intentional. `aria-label` is required, and a development warning fires when it is missing.
- Use `shape="pill"` selectively for friendly CTAs and filter actions. Standard form, toolbar, and settings actions should stay rounded.
- Polymorphism: pass `render={<a href="…" />}` (or a `<Link>`) to render as an anchor with the button's styling and behavior.
- `loading` swaps the content for a spinner and disables interaction so a double-tap can't fire twice.
- Every size ships a hidden 44×44 touch target underneath, so the tap area works on coarse pointers even for a small button.
- Labels: Title Case, verb + noun (`Deploy Project`, `Delete Member`). A lone `Submit` or `OK` tells the user nothing about what will happen.
</Callout>
