# Menu

A dropdown menu with support for items, groups, checkbox items, radio items, sub-menus, keyboard shortcuts, and separators. Built on Base UI Menu: `SubmenuRoot` handles nested submenu coordination and safePolygon hover, roving keyboard navigation and type-ahead ship in the box, and open / close uses CSS-driven `data-starting-style` / `data-ending-style` transitions.

## Usage

```tsx
import {
  Button,
  Menu,
  MenuTrigger,
  MenuPopup,
  MenuItem,
  MenuGroup,
  MenuGroupLabel,
  MenuSection,
  MenuCheckboxItem,
  MenuRadioGroup,
  MenuRadioItem,
  MenuSeparator,
  MenuShortcut,
  MenuSub,
  MenuSubTrigger,
  MenuSubPopup,
} from "@patchui/react";

// Basic menu
<Menu>
  <MenuTrigger render={<Button />}>Open Menu</MenuTrigger>
  <MenuPopup>
    <MenuItem>Profile</MenuItem>
    <MenuItem>Settings</MenuItem>
    <MenuSeparator />
    <MenuItem variant="destructive">Log out</MenuItem>
  </MenuPopup>
</Menu>

// Single-select with a trailing check
const [sortBy, setSortBy] = useState("newest");
<MenuItem selected={sortBy === "newest"} onClick={() => setSortBy("newest")}>
  Newest first
</MenuItem>

// Two-line items
<MenuItem description="Search across the workspace">
  Search
  <MenuShortcut>⌘K</MenuShortcut>
</MenuItem>

// Sections (sugar for MenuGroup + MenuGroupLabel)
<MenuSection label="Account">
  <MenuItem>Profile</MenuItem>
  <MenuItem>Settings</MenuItem>
</MenuSection>

// Comfortable density (default is compact)
<MenuPopup density="comfortable">...</MenuPopup>

// With checkbox items
const [checked, setChecked] = useState(true);
<MenuCheckboxItem checked={checked} onCheckedChange={setChecked}>
  Show Sidebar
</MenuCheckboxItem>

// With radio items
const [value, setValue] = useState("system");
<MenuRadioGroup value={value} onValueChange={setValue}>
  <MenuRadioItem value="light">Light</MenuRadioItem>
  <MenuRadioItem value="dark">Dark</MenuRadioItem>
  <MenuRadioItem value="system">System</MenuRadioItem>
</MenuRadioGroup>
```

<Callout title="Notes">
- **Trigger open state:** Button-based triggers use a variant-appropriate held fill without borrowing the text-input focus ring.
- **Keyboard navigation:** Supports arrow keys, type-ahead, Home/End, Enter/Space, and Escape.
- **Sub-menus:** Nest `MenuSub` > `MenuSubTrigger` + `MenuSubPopup` for cascading menus. Sub-menus open on hover. The sub-popup inherits the parent `MenuPopup`'s density unless overridden.
- **Portal rendering:** The popup renders outside the DOM tree via a portal, so it appears above all other content.
- **Mobile:** Below the `md` breakpoint (max-width 767px), a top-level `MenuPopup` renders as a centered card at `top-1/2 left-1/2` with `w-[calc(100vw-1rem)]` and a dimmed backdrop. `RemoveScroll` locks body scroll while open so background gestures don't leak through the backdrop. Sub-menus stay as anchored popovers so they hang off their parent row.
- **`MenuRadioItem` matches `selected`:** Radio rows render with a trailing check when they're the group's current value (same layout as `MenuItem selected`) and close the menu on click. Consistent affordance across every single-select menu.
- **Inset alignment:** Use the `inset` prop on `MenuItem`, `MenuGroupLabel`, and `MenuSubTrigger` to align text with items that have icons or check indicators.
- **Motion:** Open / close driven by Base UI's `data-starting-style` and `data-ending-style` transitions at overlay duration, so both directions animate without a JS animation runtime. Respects `prefers-reduced-motion`.
- **Foundation:** Built on Base UI Menu — `SubmenuRoot` + `SubmenuTrigger` handle nested submenu coordination and safePolygon hover; roving keyboard navigation, type-ahead, and focus scoping are handled by the primitive.
- **Opening focus:** Patch UI does not implement custom item focus or synthetic pointer behavior. Base UI keeps pointer-opened menus unselected and manages keyboard-opened menu focus natively.
- **`selected` vs `MenuCheckboxItem`:** Use `selected` for single-select "currently chosen" UI with a trailing check. Use `MenuCheckboxItem` for multi-select toggles with a leading indicator.
- **Row states:** Highlighted means temporary keyboard or pointer navigation and uses the subtle hover overlay. Selected and checked rows use the stronger selected overlay.
</Callout>
