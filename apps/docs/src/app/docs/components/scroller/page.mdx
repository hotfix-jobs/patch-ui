# Scroller

A scroll container for overflowing lists: horizontal rails, long vertical feeds, or free bidirectional canvases. Built on Base UI Scroll Area with an overlay scrollbar in vertical / bidirectional modes, and hidden in horizontal mode where the edge fade masks are the affordance.

Use for card rails (products, categories, cover art), long vertical feeds (activity logs, notifications), and free-scroll canvases (image comparisons, wide diagrams).

## Usage

```tsx
import { Scroller } from "@patchui/react";

<Scroller overflow="x" childrenContainerClassName="gap-3">
  {items.map((item) => (
    <Card key={item.id} className="shrink-0 w-64">
      {/* … */}
    </Card>
  ))}
</Scroller>
```

<Callout title="Notes">
- **Consistent item widths.** In horizontal rails, give children a fixed width and `shrink-0` so the rail snaps cleanly.
- **Edge fade.** When there's more content past an edge, that edge fades to transparent. This is the "more content" affordance in horizontal mode, and a soft cue in vertical mode alongside the scrollbar.
- **Scrollbar visibility.** Horizontal rails hide the scrollbar entirely; vertical and both-axis rails show an overlay scrollbar that fades in on hover / scroll (Base UI Scroll Area default).
- **Keyboard.** The viewport is focusable (`tabIndex={0}`) so keyboard users can arrow through the content. Children keep their DOM tab order: Tab moves through items regardless of visual scroll direction.
- **Trackpad + touch.** Native momentum scrolling; `[-webkit-overflow-scrolling:touch]` is set so iOS scrolls smoothly.
- **Not for pagination.** Reach for `Pagination` when there are discrete pages of the same dataset, or a virtualization / infinite-scroll library for append-as-you-scroll feeds. Scroller is only for content that overflows one screen.
</Callout>
