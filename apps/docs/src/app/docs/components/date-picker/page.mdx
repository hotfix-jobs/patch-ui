# DatePicker

Single or range date selection from a button trigger that opens a floating calendar. The trigger label shows the formatted selected date(s); placeholder when empty.

<Install name="date-picker" />

<ComponentPreview label="DatePicker">
  <DatePickerDemo />
</ComponentPreview>

## Usage

```tsx
import {
  DatePicker,
  DateRangePicker,
  type DateRange,
} from "@patchui/react";

// Single
const [date, setDate] = useState<Date | null>(null);
<DatePicker value={date} onValueChange={setDate} />

// Range
const [range, setRange] = useState<DateRange>({});
<DateRangePicker value={range} onValueChange={setRange} />

// Custom format
<DatePicker
  format={(d) => d.toISOString().slice(0, 10)}  // YYYY-MM-DD
  placeholder="YYYY-MM-DD"
/>

// Constraints
<DatePicker
  fromDate={new Date()}  // no past dates
  disabledDates={(d) => d.getDay() === 0}  // disable Sundays
/>

// Sizes match Button
<DatePicker size="sm" />
<DatePicker size="md" />  // default
<DatePicker size="lg" />
```

### DateRangePicker

Same props as DatePicker, except:

- `value` / `defaultValue` / `onValueChange` use `DateRange` (`{ from?, to? }`)
- Auto-closes when both `from` and `to` are picked (not on first click)
- Label format: `"Jan 1, 2026 – Feb 5, 2026"` (or just the start date if only `from` is set)

<Callout title="Notes">
- **Composition over abstraction:** DatePicker is a thin shell around a button trigger and `Calendar`. If you need additional UI inside the popup (preset shortcuts like "Last 7 days", a time picker, footer actions), drop the wrapper and compose a `Menu` or `Dialog` around `Calendar` directly.
- **Auto-close:** DatePicker closes when a date is selected. DateRangePicker stays open after the first click so the user can pick the end date, then closes once both are set.
- **Calendar feature parity:** All `Calendar` props (`fromDate`, `toDate`, `disabledDates`, `weekStartsOn`, `defaultMonth`) pass through to the inner calendar.
</Callout>
