# EmptyState

A quiet "no results" or "no data yet" placeholder. It provides structure for an optional icon, title, description, primary action, and supporting content without adding its own card chrome.

## Usage

```tsx
import { EmptyState, Button } from "@patchui/react";
import { ChartBar, ArrowSquareOut } from "@phosphor-icons/react/dist/ssr";
// Basic
<EmptyState
  title="No results match your filters"
  description="Try removing a filter, or clear all to see every result."
  icon={<BarChart3 />}
  action={<Button variant="secondary">Clear Filters</Button>}
/>

// With a secondary Learn more link
<EmptyState
  title="Title"
  description="This should detail the actions you can take on this screen, as well as why it's valuable."
  icon={<BarChart3 />}
  action={<Button variant="secondary">Primary Action</Button>}
>
  <a href="#" className="inline-flex items-center gap-1 text-small text-ink hover:text-ink">
    Learn more
    <ExternalLink className="size-3.5" />
  </a>
</EmptyState>
```

<Callout title="Notes">
- EmptyState adds no background, border, shadow, or entrance animation. Its parent owns the surrounding surface.
- The icon is muted and rendered at 32px. It does not sit inside a decorative container.
- Cap at one primary CTA in `action`. Use `children` for a secondary link (typical: "Learn more" with an external-link icon) or supporting content.
- Content guidance: title is Title Case (`No Logs Match Your Filter`); description is sentence case and adds new information, not a rephrasing of the title. CTA labels are Title Case Verb + Noun (`Deploy Project`, `Browse Items`), never `Get Started` or `Continue`.
- For an async filter change, wrap the region in `aria-live="polite"` so screen readers announce the new state.
- Wraps well inside a `<Card>` for the settings-page or dashboard empty section look.
</Callout>
