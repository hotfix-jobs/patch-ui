# EmptyState

"No results" / "no data yet" placeholder. Hairline-bordered icon container, title, optional description and action. Fades in on mount so the transition from loading or list-with-data feels intentional.

<Install name="empty-state" />

<ComponentPreview label="EmptyState">
  <EmptyStateDemo />
</ComponentPreview>

## Usage

```tsx
import { EmptyState, Button } from "@patchui/react";
import { Search } from "lucide-react";

<EmptyState
  title="No jobs match those filters"
  description="Try removing a filter, or reset to see everything."
  icon={<Search className="size-4" />}
  action={<Button variant="primary">Browse jobs</Button>}
/>
```

<Callout title="Notes">
- **Motion on mount:** Fades in with a subtle `opacity 0 → 1` + 8px y-translate on mount via `motion`. Respects `prefers-reduced-motion`. Makes transitions from skeleton or list-with-data into the empty state feel intentional rather than abrupt.
- **Icon container:** Hairline-bordered 36×36 box with editorial radius. The icon inherits `--patch-text-tertiary` color; SVG inside scales automatically.
- **Routing:** Pass any node as `action`. For framework integration (Next.js Link, React Router, etc.), wrap your `<Button>` inside the router's Link primitive — Patch stays framework-agnostic.
</Callout>
