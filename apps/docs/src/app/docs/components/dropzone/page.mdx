# Dropzone

Drag-and-drop file upload with a click-to-browse fallback. Renders a preview row per file with a remove × button. Pure UI primitive — consumers wire `onValueChange` to their upload pipeline.

Use for any file-upload UX: avatars, attachments, document uploads, resume parsers, batch image imports.

## Usage

```tsx
import { Dropzone } from "@patchui/react";

const [files, setFiles] = useState<File[]>([]);

<Dropzone
  value={files}
  onValueChange={setFiles}
  accept="image/*,application/pdf"
  maxSize={5 * 1024 * 1024}  // 5 MB per file
  maxFiles={10}
  title="Drop files here or click to browse"
  description="Up to 5 MB, images or PDFs"
/>

// Single file (no list)
<Dropzone multiple={false} showFileList={false} />

// Custom validation
<Dropzone
  validate={(file, current) => {
    if (file.size > 2 * 1024 * 1024) return "File too large";
    if (current.some((f) => f.name === file.name)) return "Duplicate";
    return true;
  }}
/>

// Custom file row
<Dropzone
  renderFile={(file, i, onRemove) => (
    <div className="my-custom-row">
      <img src={URL.createObjectURL(file)} />
      <button onClick={onRemove}>×</button>
    </div>
  )}
/>
```

<Callout title="Notes">
- **Native file input:** The drop area is a `<button>` (keyboard accessible — Enter / Space open the file picker). The underlying `<input type="file">` is `sr-only` and triggered via click.
- **Drag-over feedback:** `data-drag-over` is set on the drop area while files hover. Default styling: border darkens to `--patch-text` and bg shifts to `--patch-accent`.
- **Upload progress per file:** Wire `Progress` into your `renderFile` to show per-file upload status while your pipeline runs.
- **No built-in upload:** Dropzone collects files; you call `fetch` / mutate / your storage SDK from `onValueChange`. Keeps the primitive framework-agnostic.
</Callout>
