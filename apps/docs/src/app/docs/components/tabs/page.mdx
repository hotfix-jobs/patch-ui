# Tabs

Compound component built on Base UI Tabs. Two variants: `underline` (default) draws a single sliding underline; `pill` slides a rounded background between active tabs. Both indicators animate spring-eased between positions via Base UI's `Tabs.Indicator` + the `--active-tab-*` CSS variables it populates. The `pill` variant works in both orientations, so one component can render a vertical sidebar on desktop and a horizontal strip on mobile.

## Usage

```tsx
import { Tabs, TabsList, TabsTrigger, TabsPanel } from "@patchui/react";

// Content tabs (underline)
<Tabs defaultValue="overview">
  <TabsList>
    <TabsTrigger value="overview">Overview</TabsTrigger>
    <TabsTrigger value="settings">Settings</TabsTrigger>
  </TabsList>
  <TabsPanel value="overview">...</TabsPanel>
  <TabsPanel value="settings">...</TabsPanel>
</Tabs>

// Nav-style, responsive: vertical pill on desktop, horizontal on mobile
<Tabs variant="pill" orientation="vertical" defaultValue="profile">
  <TabsList>
    <TabsTrigger value="profile">Profile</TabsTrigger>
    <TabsTrigger value="billing">Billing</TabsTrigger>
  </TabsList>
</Tabs>
```

## Components

- `Tabs` - root; accepts `variant`, `orientation`, `defaultValue` or controlled `value`/`onValueChange`
- `TabsList` - container for triggers; renders the sliding indicator (underline OR pill background)
- `TabsTrigger` - individual tab; for pill variant, sits above the sliding indicator via z-index
- `TabsPanel` - matched panel for a trigger value

<Callout title="Notes">
- **Sliding indicator:** Both variants share the same motion `layoutId` infrastructure. Underline draws a 2px `gray-1000` bar under the active tab; pill paints a `gray-alpha-100` wash behind the active tab. Motion springs the indicator between active tabs.
- **Underline vs pill:** Use `underline` for switching content panels; use `pill` for navigation-style tabs (settings sections, sidebar nav) where a filled active state reads better.
- **Responsive nav:** With `variant="pill"`, render `orientation="vertical"` on larger screens and horizontal on small screens — the sliding indicator works in both.
- **Keyboard:** Triggers are arrow-key navigable (Base UI) and show the standard focus outline.
</Callout>
