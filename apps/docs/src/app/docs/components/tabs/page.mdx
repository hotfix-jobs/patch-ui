# Tabs

A quiet selected-navigation control built on Base UI Tabs. Triggers are borderless and use the shared interaction overlay for hover and selection. Horizontal and vertical orientations share the same state treatment.

## Usage

```tsx
import { Tabs, TabsList, TabsTrigger, TabsPanel } from "@patchui/react";

// Horizontal (default)
<Tabs defaultValue="overview">
  <TabsList>
    <TabsTrigger value="overview">Overview</TabsTrigger>
    <TabsTrigger value="settings">Settings</TabsTrigger>
  </TabsList>
  <TabsPanel value="overview">...</TabsPanel>
  <TabsPanel value="settings">...</TabsPanel>
</Tabs>

// Vertical: list on the left, panel on the right
<Tabs orientation="vertical" defaultValue="profile">
  <TabsList className="w-44">
    <TabsTrigger value="profile">Profile</TabsTrigger>
    <TabsTrigger value="billing">Billing</TabsTrigger>
  </TabsList>
  <TabsPanel value="profile">...</TabsPanel>
  <TabsPanel value="billing">...</TabsPanel>
</Tabs>
```

## Components

- `Tabs` - root; accepts `orientation`, `defaultValue` or controlled `value`/`onValueChange`
- `TabsList` - borderless container for triggers
- `TabsTrigger` - individual tab; accepts `icon`, `badge` (numeric badges hide at 0), `tooltip` (for disabled), and carries `data-[active]` for active-state styling
- `TabsPanel` - matched panel for a trigger value; horizontally adds `pt-6`, vertically becomes the flex-1 column beside the list

<Callout title="Notes">
- **States:** Hover and stored selection use `layer-hover`. Text weight and ARIA state distinguish the selected tab.
- **Keyboard:** Base UI provides roving tabindex, arrow-key navigation, and Home / End. Keyboard focus uses the compact neutral indicator shared by non-editable controls.
</Callout>
