# Sheet

An edge-anchored panel that slides in from the top, right, bottom, or left. Non-modal by default: the page underneath stays interactive so consumers can inspect one entity while browsing the list. Pass `modal` to darken the backdrop and block the page.

## Usage

```tsx
import {
  Button,
  Sheet,
  SheetTrigger,
  SheetContent,
  SheetHeader,
  SheetTitle,
  SheetDescription,
  SheetBody,
  SheetFooter,
  SheetClose,
} from "@patchui/react";

<Sheet>
  <SheetTrigger render={<Button variant="secondary">Open profile</Button>} />
  <SheetContent>
    <SheetHeader>
      <SheetTitle>Profile</SheetTitle>
      <SheetDescription>Details for the selected member.</SheetDescription>
      <SheetClose />
    </SheetHeader>
    <SheetBody>
      {/* form / inspector content */}
    </SheetBody>
    <SheetFooter>
      <SheetClose render={<Button variant="secondary">Cancel</Button>} />
      <SheetClose render={<Button variant="primary">Save</Button>} />
    </SheetFooter>
  </SheetContent>
</Sheet>
```

<Callout title="Notes">
- **Non-modal by default.** No backdrop, page underneath keeps its own interactivity and scrolls independently. This matches the Vercel Geist default: sheets are for persistent inspectors alongside the page, not confirmations that require full attention.
- **Outside-click doesn't dismiss.** Only Escape and an explicit `<SheetClose>` close the sheet. Prevents accidental dismissal when the sheet holds an unsaved form.
- **Modal mode.** Pass `modal` to `SheetContent` when the sheet configures the view underneath (filters, region picker) — you want the page paused while the choice is made.
- **Side matters.** Right for detail inspectors, left for navigation, bottom for mobile-first pickers. Keep the direction consistent across similar patterns in the app.
- **Focus return.** Focus returns to the trigger on close.
</Callout>
