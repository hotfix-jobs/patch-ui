# Sheet

An edge-anchored panel that slides in from the top, right, bottom, or left. It uses one rounded outer boundary and shadow, with borderless internal regions for a quieter surface.

## Usage

```tsx
import {
  Button,
  Sheet,
  SheetTrigger,
  SheetContent,
  SheetHeader,
  SheetTitle,
  SheetDescription,
  SheetBody,
  SheetFooter,
  SheetClose,
} from "@patchui/react";

<Sheet>
  <SheetTrigger render={<Button variant="secondary">Open profile</Button>} />
  <SheetContent>
    <SheetHeader>
      <SheetTitle>Profile</SheetTitle>
      <SheetDescription>Details for the selected member.</SheetDescription>
    </SheetHeader>
    <SheetBody>
      {/* form / inspector content */}
    </SheetBody>
    <SheetFooter>
      <SheetClose render={<Button variant="secondary">Cancel</Button>} />
      <SheetClose render={<Button variant="primary">Save</Button>} />
    </SheetFooter>
  </SheetContent>
</Sheet>
```

<Callout title="Notes">
- **One surface.** The outer panel provides the boundary and elevation. Header, body, and footer remain visually quiet inside it.
- **Outside-click, Escape, and `<SheetClose>` all dismiss.** For sheets holding unsaved input, wrap Save/Cancel in `SheetClose` and gate the dismiss with a confirmation step upstream.
- **Side matters.** Right for detail inspectors, left for navigation, bottom for mobile-first pickers. Keep the direction consistent across similar patterns in the app.
- **Focus return.** Focus returns to the trigger on close.
</Callout>
