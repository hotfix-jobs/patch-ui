# Pagination

A minimal page navigator: prev/next icon buttons around an inline list of page numbers. Numbers read as quiet navigation chrome rather than a row of stamped buttons; the active page gets bold text and a hairline underline. Renders nothing when there is only one page.

## Usage

```tsx
import { useState } from "react";
import { Pagination } from "@patchui/react";

const [page, setPage] = useState(1);

// Basic
<Pagination page={page} totalPages={10} onPageChange={setPage} />

// Wider sibling window (active ±2 instead of ±1)
<Pagination page={page} totalPages={20} onPageChange={setPage} siblingCount={2} />

// With First / Last jump buttons
<Pagination page={page} totalPages={50} onPageChange={setPage} showFirstLast />

// Compact — just prev/next + "Page X of Y", no numbered list
<Pagination page={page} totalPages={10} onPageChange={setPage} compact />

// Loading state during async page transitions
<Pagination
  page={page}
  totalPages={10}
  onPageChange={async (next) => {
    setLoading(true);
    await fetchPage(next);
    setPage(next);
    setLoading(false);
  }}
  loading={loading}
/>
```

<Callout title="Notes">
- **Tabular numerals:** Page numbers use tabular figures so digits stay column-aligned regardless of width.
- **Truncation:** Always shows the first and last page; ellipsis markers fill gaps when the active window doesn't reach the edges. Widen the window with `siblingCount={2}` or larger.
- **Tap targets:** Numbered buttons are `min-w-[32px]` with `px-2 py-1.5` padding — a proper 32×32 touch target while keeping the inline-text feel.
- **Press feedback:** Numbers scale to 95% on `active:` (matching `Button`), so clicks have tactile confirmation.
- **Active semantics:** The active page button gets `aria-current="page"` and `aria-disabled` — semantically it isn't navigable since you're already there. Screen readers announce it as the current page.
- **Loading state:** When `loading={true}`, every control is disabled, the nav fades, and `aria-busy="true"` signals to assistive tech that a transition is in progress.
- **Keyboard:** All controls are Tab-focusable with the standard focus outline. Arrow keys aren't bound (they'd conflict with horizontal scroll containers); use Tab + Enter.
</Callout>
