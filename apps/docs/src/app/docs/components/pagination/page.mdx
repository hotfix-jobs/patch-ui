# Pagination

A grouped bar of page numbers between prev/next arrows, enclosed in a single hairline pill. First and last pages are always rendered so crawlers discover the whole series; an ellipsis fills the gaps for long ranges. Renders nothing when there is only one page.

## Usage

```tsx
import { useState } from "react";
import { Pagination } from "@patchui/react";

const [page, setPage] = useState(1);

// Client-side (dashboards, auth-gated pages)
<Pagination page={page} totalPages={12} onPageChange={setPage} />

// Loading state during async page transitions
<Pagination
  page={page}
  totalPages={12}
  onPageChange={async (next) => {
    setLoading(true);
    await fetchPage(next);
    setPage(next);
    setLoading(false);
  }}
  loading={loading}
/>
```

## SEO-friendly server-rendered pages

For indexable content (product catalogs, directories, blog archives), pass an `href` builder. Every control renders as a real `<a href>` with no JavaScript required, so it remains crawlable and works with SSR or RSC.

```tsx
// Server component
<Pagination
  page={currentPage}
  totalPages={totalPages}
  href={(p) => (p === 1 ? "/products" : `/products?page=${p}`)}
/>
```

The `p === 1` special-case keeps the canonical URL for the first page as the bare route (`/products` rather than `/products?page=1`), so the same page is not indexed under two URLs.

<Callout title="Notes">
- **First and last always visible:** Even in long ranges, page 1 and the final page always render. Ellipsis fills gaps in between. This is deliberate for crawler discovery: with `href`, every enumerated page URL becomes a real anchor Googlebot can follow.
- **Tabular numerals:** Page numbers use tabular figures so digits stay column-aligned regardless of width.
- **Active semantics:** The active page cell gets `aria-current="page"` and reads with a subtle `surface-2` lift (matching how Menu and Combobox items highlight their active row), not a shouty solid fill.
- **Loading state:** When `loading={true}`, every control is disabled, the nav fades, and `aria-busy="true"` signals to assistive tech that a transition is in progress.
- **Keyboard:** All controls are Tab-focusable. Use Tab + Enter or Space to navigate; arrow keys are intentionally unbound so the control composes inside horizontal scroll containers.
</Callout>
