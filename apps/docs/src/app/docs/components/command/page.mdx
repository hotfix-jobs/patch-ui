# Command

A command palette built on Base UI's Autocomplete (inline mode: the list is always visible, no positioned dropdown). Type to filter, arrow keys to move, Enter to select. Pass `items` for built-in filtering, or `filteredItems` to drive results from your own search index. Wrap in `CommandDialog` for a Cmd+K palette.

## Usage

```tsx
import {
  CommandDialog,
  CommandInput,
  CommandList,
  CommandEmpty,
  CommandItem,
  CommandSection,
} from "@patchui/react";

const items = ["New file", "Open file", "Save"];

<CommandDialog open={open} onOpenChange={setOpen} items={items}>
  <CommandInput placeholder="Type a command..." />
  <CommandList>
    <CommandEmpty>No results found.</CommandEmpty>
    <CommandCollection>
      {(item) => (
        <CommandItem key={item} value={item} onClick={() => run(item)}>
          {item}
        </CommandItem>
      )}
    </CommandCollection>
  </CommandList>
</CommandDialog>

// With a trailing "currently chosen" check
<CommandItem selected={value === item} value={item} onClick={...}>
  {item}
</CommandItem>

// Two-line item with a description
<CommandItem description="Switch between light and dark" value="Toggle theme">
  Toggle theme
</CommandItem>

// Sections (sugar for CommandGroup + CommandGroupLabel)
<CommandSection label="Navigate">
  <CommandItem>Go to dashboard</CommandItem>
  <CommandItem>Go to projects</CommandItem>
</CommandSection>

// Comfortable density (larger items + body type size)
<CommandList density="comfortable">...</CommandList>
```

For grouped results, wrap items in `CommandSection` (or `CommandGroup` + `CommandGroupLabel`) + `CommandCollection`. To drive results from a fuzzy index, hold the query in state via the input and pass your matched array as `filteredItems` instead of `items`.

## Components

- `Command` - root (Base UI Autocomplete in inline mode); takes `items` or `filteredItems`
- `CommandDialog` - `Command` inside a `Modal` for a Cmd+K palette
- `CommandInput` - search input with a leading icon
- `CommandList` - the results list; supports `density`
- `CommandItem` - a selectable result; supports `selected` and `description`
- `CommandEmpty` - shown when nothing matches
- `CommandSection` - labeled group with consistent spacing, sugar for `CommandGroup` + `CommandGroupLabel`
- `CommandGroup` / `CommandGroupLabel` / `CommandCollection` - low-level grouped sections
- `CommandSeparator` - a divider between groups

## Keyboard hints (`Kbd`)

Shortcut chips shown in the `suffix` slot of a `CommandItem` are rendered with the `Kbd` primitive. It handles platform glyph switching for you (`⌘` on Mac, `Ctrl` on Windows / Linux) and composes multiple modifiers into one chip.

```tsx
import { Kbd } from "@patchui/react";

<CommandItem
  suffix={<Kbd meta>N</Kbd>}
  onClick={() => run("new-file")}
>
  New file
</CommandItem>

// Multiple modifiers compose in order
<Kbd meta shift>P</Kbd>  // ⇧⌘P on Mac, Ctrl+Shift+P elsewhere
<Kbd ctrl>K</Kbd>        // ⌃K on Mac, Ctrl+K elsewhere
```

Props: `meta` (⌘ / Ctrl), `ctrl` (⌃ / Ctrl), `alt` (⌥ / Alt), `shift` (⇧), plus a `size` (`sm` | `md`) and an optional `onClick` — passing a handler renders it as a `<button>` with hover state, useful for a clickable Esc chip in an input's `action` slot.

<Callout title="Notes">
- **Built on Base UI Autocomplete** - filtering, roving focus, type-ahead, and ARIA are handled for you; no extra command library.
- **Bring your own search:** pass `filteredItems` to render results from a fuzzy index (this docs site does exactly that for its Cmd+K search).
- **Keyboard:** type to filter, Up/Down to move, Enter to select the highlighted item, Esc to close.
- **`selected` vs filtering:** `selected` is purely visual (renders the trailing check). It does not affect filtering or focus. Use it to mark a "currently chosen" item in single-select pickers.
</Callout>
