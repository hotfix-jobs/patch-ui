# Card

A surface primitive for search, list, and content cards. Use the default `surface`, choose `outlined` when a boundary carries meaning, or `elevated` when the object needs shadow separation.

Card is intentionally content-agnostic. For structured settings-page panels with labeled rows, reach for `Section`.

## Usage

```tsx
import { Card } from "@patchui/react";

// Default content surface
<Card className="p-4">A clear content card.</Card>

// Explicit boundary
<Card variant="outlined" className="p-4">Administrative content.</Card>

// Elevated surface
<Card variant="elevated" className="p-4">
  A card that reads as a distinct object.
</Card>

// Interactive (elevation + focus ring)
<Card interactive className="p-4">Hover me</Card>

// Selected state (multi-pick galleries, plan comparison)
<Card selected className="p-4">
  Selected plan
</Card>
```

<Callout title="Notes">
- The three variants express surface intent: standard content, explicit boundary, or elevation.
- `interactive` adds focus treatment and hover elevation without decorative border chrome.
- Card does not own layout or dividers. Compose those with `flex`, `grid`, and border utilities at the call site.
- Structure content directly inside Card with divs and text tokens (`text-title3`, `text-small`). Use `<Section>` when the content needs named structural header, content, or footer slots.
- **Migration from prior API:** `flat` becomes `surface`, `hoverable` becomes `interactive`, and layout or divider props move to caller composition.
</Callout>
