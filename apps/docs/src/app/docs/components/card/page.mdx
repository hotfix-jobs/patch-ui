# Card

A surface primitive. Bordered, rounded, tokenized. Compose with boolean props (`border`, `hoverable`, `shadow`, `secondary`, `borderBetween`, `direction`) to fit the surface you need.

Card is intentionally content-agnostic. For structured header/content/footer patterns (settings pages, plan pickers), reach for `Section`.

## Usage

```tsx
import { Card } from "@patchui/react";

// Default — bordered
<Card className="p-4">A simple bordered card</Card>

// Bare (opt out of the border)
<Card border={false} className="p-4">Just a subtle bg</Card>

// Hoverable (bg + border shift on hover, focus ring)
<Card hoverable className="p-4">Hover me</Card>

// Elevated (shadow-card)
<Card shadow className="p-4">Elevated</Card>

// Secondary surface (--fill-1 fill) for nested cards
<Card secondary className="p-4">Nested surface</Card>

// List container with dividers between direct children
<Card borderBetween>
  <div className="p-4">Option 1</div>
  <div className="p-4">Option 2</div>
  <div className="p-4">Option 3</div>
</Card>

// Row direction with vertical dividers
<Card borderBetween direction="row">
  <div className="p-4 flex-1">Left</div>
  <div className="p-4 flex-1">Right</div>
</Card>

// Selected state (multi-pick galleries, plan comparison)
<Card selected className="p-4">
  Selected plan
</Card>
```

<Callout title="Notes">
- Card composes with booleans, not variants. Mix `border` + `hoverable` + `shadow` + `secondary` to build the exact surface you need.
- The default Card (no props) is a rounded surface with `--layer-1` fill so it elevates over the page. Add `border` to give it an outline.
- `hoverable` makes the card interactive (cursor, hover state, focus ring). Wrap in a `<Link>` or handle `onClick` for actual navigation.
- `borderBetween` turns Card into a list container: dividers between direct children handle themselves. Combine with `direction="row"` for horizontal splits.
- Structure your content directly inside Card with divs and text tokens (`text-title3`, `text-small`). For a fully structured settings-page pattern (title, subtitle, footer with status + actions), use `<Section>` instead.
- **Migration from v1:**
  - `variant="default"` → no prop.
  - `variant="elevated"` → `secondary`.
  - `variant="interactive"` → `hoverable border`.
  - `variant="ghost"` → drop; use `className` for the specific styling you need.
  - `CardHeader` / `CardTitle` / `CardDescription` / `CardContent` / `CardFooter` / `CardAction` / `CardMeta` sub-components removed. Migrate to `<Section>` for structured settings-page cards, or hand-roll headers with divs + text tokens for one-off patterns.
</Callout>
