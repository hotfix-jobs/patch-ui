# Card

A composable content surface with a clear outlined default, optional elevation, consistent internal structure, and semantic actionable composition.

Card remains content-agnostic. Its structural parts only standardize spacing and typography. For settings panels with labeled rows, reach for `Section`.

## Usage

```tsx
import Link from "next/link";
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
} from "@patchui/react";

// Default outlined card
<Card>
  <CardHeader>
    <div>
      <CardTitle>Activity</CardTitle>
      <CardDescription>Changes from the last seven days.</CardDescription>
    </div>
  </CardHeader>
  <CardContent>Three updates were published.</CardContent>
  <CardFooter divided>Updated recently</CardFooter>
</Card>

// Quiet surface
<Card variant="surface">
  <CardContent>Grouped content.</CardContent>
</Card>

// Elevated object
<Card variant="elevated">
  <CardContent>Prominent content.</CardContent>
</Card>

// Actionable cards require link or button semantics
<Card actionable render={<Link href="/resources" />}>
  <CardContent>Browse resources</CardContent>
</Card>

// Visual selection only. Supply semantics appropriate to the parent widget.
<Card selected>
  <CardContent>Selected item</CardContent>
</Card>
```

<Callout title="Notes">
- `outlined` is the default because application cards usually benefit from a clear object boundary.
- `actionable` emphasizes the existing border instead of filling the complete surface. Pair it with `outlined` or `elevated` so the pointer state has a visible boundary to strengthen.
- Structural parts are independent render-prop primitives. They share no context and can be omitted or reordered.
- `actionable` is visual behavior, not semantics. Pair it with `render={<Link />}` or `render={<button />}`.
- `selected` does not emit `aria-selected`. Add the ARIA state required by the surrounding listbox, grid, or selection control.
- Use Section when content is primarily a labeled panel or collection of divided rows.
</Callout>
