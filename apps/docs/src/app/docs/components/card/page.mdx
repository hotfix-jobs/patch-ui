# Card

A surface primitive. Rounded, tokenized, with two visual treatments. Pick `flat` (default) for the quiet frame used across search, list, and content surfaces, or `elevated` when a surface should read as a distinct object.

Card is intentionally content-agnostic. For structured settings-page panels with labeled rows, reach for `Section`.

## Usage

```tsx
import { Card } from "@patchui/react";

// Default — flat frame with hairline border
<Card className="p-4">A quiet, transparent card.</Card>

// Elevated — bg-layer-1 + soft edge + light-mode shadow
<Card variant="elevated" className="p-4">
  A card that reads as a distinct object.
</Card>

// Hoverable (border emphasis + focus ring)
<Card hoverable className="p-4">Hover me</Card>

// List container with dividers between direct children
<Card borderBetween>
  <div className="p-4">Option 1</div>
  <div className="p-4">Option 2</div>
  <div className="p-4">Option 3</div>
</Card>

// Row direction with vertical dividers
<Card borderBetween direction="row">
  <div className="p-4 flex-1">Left</div>
  <div className="p-4 flex-1">Right</div>
</Card>

// Selected state (multi-pick galleries, plan comparison)
<Card selected className="p-4">
  Selected plan
</Card>
```

<Callout title="Notes">
- Two treatments only: `flat` and `elevated`. `flat` sits transparent on the page — use it for list rows, search results, filter chips, anything that should read as content, not chrome. `elevated` fills with `--layer-1` and gains `--hairline-soft` + `shadow-card` for auth cards, marketing lifts, and any surface that should distinctly separate from the page.
- `hoverable` is a quiet treatment: cursor + focus ring + `hover:border-hairline-strong`. No bg shift — keeps content cards from feeling like buttons.
- `borderBetween` turns Card into a list container: dividers between direct children handle themselves. Combine with `direction="row"` for horizontal splits.
- Structure your content directly inside Card with divs and text tokens (`text-title3`, `text-small`). For a fully structured settings-page pattern (title, subtitle, footer with status + actions), use `<Section>` instead.
- **Migration from prior API:**
  - `border={false}` → `variant="flat"` (this is now the default anyway).
  - `shadow` is still a prop; it just now composes with `variant`. `elevated` implies `shadow` unless you pass `shadow={false}`. Passing `shadow` on a `flat` card lifts the transparent frame.
  - `secondary` → drop; use `className` for a specific fill if you need it.
  - The old bg-shift `hoverable` treatment is gone — hover is now border-only.
</Callout>
