# Card

A container for grouping related content and actions. Boolean composition (`border` / `hoverable` / `shadow` / `secondary` / `borderBetween` / `direction`) — mix and match to fit the surface. Compound sub-components (`CardHeader`, `CardTitle`, `CardDescription`, `CardAction`, `CardContent`, `CardFooter`, `CardMeta`) available for structured layouts.

## Usage

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
  Button,
} from "@patchui/react";

// Basic card — just a subtle bg
<Card className="p-4">
  <p>A simple card</p>
</Card>

// With border
<Card border className="p-4">
  <p>Bordered card</p>
</Card>

// Hoverable (bg + border shift on hover)
<Card border hoverable className="p-4">
  <p>Hover me</p>
</Card>

// Secondary surface (gray-100 fill)
<Card secondary className="p-4">
  <p>Nested surface</p>
</Card>

// With shadow
<Card border shadow className="p-4">
  <p>Elevated</p>
</Card>

// List container with dividers between children
<Card border borderBetween>
  <div className="p-4">Option 1</div>
  <div className="p-4">Option 2</div>
  <div className="p-4">Option 3</div>
</Card>

// Row direction with vertical dividers
<Card border borderBetween direction="row">
  <div className="p-4 flex-1">Left</div>
  <div className="p-4 flex-1">Right</div>
</Card>

// Structured layout with sub-components
<Card border>
  <CardHeader>
    <CardTitle>Project Settings</CardTitle>
    <CardDescription>Manage your project configuration.</CardDescription>
  </CardHeader>
  <CardContent>
    <p>...</p>
  </CardContent>
  <CardFooter>
    <Button size="sm">Save</Button>
  </CardFooter>
</Card>

// Selected state (multi-pick galleries, plan comparison)
<Card border selected>...</Card>
```

### CardHeader / CardTitle / CardDescription / CardAction / CardContent / CardFooter

Structured layout helpers. Use when the card has distinct sections (title area, body, footer with actions). Skip them entirely for simple cards — just pass `className="p-4"` and put content inline.

### CardMeta

Attribution row for the bottom of a card: `icon`, `primary`, `secondary`, and optional `action`. Handy for author lines ("Jane Doe · Product Designer") and provenance stamps.

<Callout title="Notes">
- Card composes with booleans, not variants. Mix `border` + `hoverable` + `shadow` + `secondary` to build the exact surface you need.
- The default Card (no props) is a rounded surface with `background-100` fill. Add `border` to give it an outline.
- `hoverable` makes the card interactive (cursor, hover state, focus ring). Wrap in a `<Link>` or handle `onClick` for actual navigation.
- `borderBetween` turns the card into a list container — dividers between direct children handle themselves. Combine with `direction="row"` for horizontal splits.
- Sub-components are opt-in. For a simple `<Card className="p-4">Hello</Card>` you don't need `CardHeader` at all.
- **Migration from v1:** `variant="default"` → no prop. `variant="elevated"` → `secondary`. `variant="interactive"` → `hoverable border`. `variant="ghost"` → drop, use `className` for the specific styling you need.
</Callout>
