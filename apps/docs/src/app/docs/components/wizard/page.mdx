# Wizard

Chrome for a sequential multi-step flow. Owns a top bar with a dot progress indicator, an optional dismiss button, a centered max-width container, and a slide-from-right transition between steps. Common uses: onboarding, signup, multi-step forms.

Wizard does not own step state, validation, API calls, or step titles. The parent drives the `step` prop. Each step component renders its own heading, fields, and continue button.

## Usage

```tsx
import { useState } from "react";
import { Wizard } from "@patchui/react";

function MyFlow() {
  const [step, setStep] = useState(0);

  return (
    <Wizard
      step={step}
      totalSteps={4}
      onDismiss={() => router.push("/")}
    >
      {step === 0 && <StepOne onAdvance={() => setStep(1)} />}
      {step === 1 && <StepTwo onAdvance={() => setStep(2)} />}
      {step === 2 && <StepThree onAdvance={() => setStep(3)} />}
      {step === 3 && <StepFour />}
    </Wizard>
  );
}
```

## Anatomy

```tsx
<Wizard>
  <div data-slot="wizard">
    <div data-slot="wizard-bar">
      <div data-slot="wizard-progress" role="progressbar">
        <span data-state="past" />
        <span data-state="current" />
        <span data-state="future" />
      </div>
      <button data-slot="wizard-dismiss" aria-label="Close" />
    </div>
    <motion.div data-slot="wizard-step">{children}</motion.div>
  </div>
</Wizard>
```

<Callout title="Notes">
- **Dismiss is contextual:** pass `onDismiss` for overlay contexts (Sheet, Dialog) or when the flow is genuinely skippable. On a dedicated `/onboarding` page route, omit it (no X renders) — the user committed by arriving, and "back" usually belongs to the browser, not the wizard.
- **Forward only:** there's no back button. Each step is treated as a commit point; if you need bidirectional navigation, render a back affordance inside the step content and manage `step` from the parent.
- **State lives outside:** step index, validation, API calls, and analytics events all belong to the parent. Wizard is pure chrome.
- **Step keying:** Wizard uses `step` as the AnimatePresence key, so the active step component remounts every time the index changes. Local form state inside a step will reset on advance.
- **Motion:** slide-from-right (24px) with opacity (220ms), respects `prefers-reduced-motion` by collapsing to a fade.
- **Dismiss:** the X is purely a handler. Use it for skip, route-back, or modal close depending on context.
- **A11y:** progress dots are wrapped in `role="progressbar"` with `aria-valuemin` / `aria-valuenow` / `aria-valuemax`. Each dot carries `data-state="past" | "current" | "future"` for styling hooks.
</Callout>
