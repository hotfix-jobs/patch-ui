# Progress

Determinate or indeterminate progress bar. Use for upload progress, multi-step forms, "X of Y complete", or any time you need a visual indicator of completion. For unbounded loading, prefer `Spinner`; for bounded operations, use Progress.

## Usage

```tsx
import { Progress } from "@patchui/react";

// Determinate (0-100)
<Progress value={40} />

// Custom max
<Progress value={3} max={5} label="Step 3 of 5" />

// Variants
<Progress value={70} variant="warning" />
<Progress value={92} variant="danger" />
<Progress value={100} variant="success" />

// Sizes
<Progress value={50} size="sm" />  // 4px tall
<Progress value={50} size="md" />  // 6px (default)
<Progress value={50} size="lg" />  // 10px tall

// Indeterminate
<Progress value={null} />
```

<Callout title="Notes">
- **Motion:** The fill width animates via a spring transition (`stiffness 200, damping 25`) — smooth when the value changes incrementally. Indeterminate mode sweeps a 1/3-width pill from off-screen-left to off-screen-right on an infinite loop.
- **A11y:** Renders `role="progressbar"` with `aria-valuemin`, `aria-valuemax`, and `aria-valuenow` for determinate mode. For indeterminate, those attrs are omitted (per WAI-ARIA spec). Pass a meaningful `label`.
- **Reduced motion:** With `prefers-reduced-motion`, the fill snaps to the new value instead of springing, and indeterminate mode parks the pill at 0% instead of sweeping.
</Callout>
