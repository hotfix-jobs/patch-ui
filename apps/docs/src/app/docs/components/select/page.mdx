# Select

A native `<select>` wrapped in styled chrome. Uses the OS-native dropdown, which behaves correctly on every platform, especially mobile where the native picker is far better than any custom popup.

## Usage

```tsx
import { Select } from "@patchui/react";

<Select
  label="Framework"
  placeholder="Select a framework…"
  value={framework}
  onChange={(e) => setFramework(e.target.value)}
>
  <option value="react">React</option>
  <option value="vue">Vue</option>
  <option value="svelte">Svelte</option>
</Select>

// With grouped options
<Select label="Team member" placeholder="Assign to…">
  <optgroup label="Design">
    <option value="ana">Ana</option>
    <option value="ben">Ben</option>
  </optgroup>
  <optgroup label="Engineering">
    <option value="chris">Chris</option>
    <option value="dana">Dana</option>
  </optgroup>
</Select>
```

<Callout title="Notes">
- **Native picker:** The dropdown itself is drawn by the OS (macOS system menu, iOS wheel picker, Android bottom sheet, Windows dropdown). We don't style it — that's the whole point.
- **Rich options:** Native `<option>` accepts text only. If you need icons, descriptions, or inline actions in the list, reach for `Combobox` instead.
- **Form integration:** Since this is a real `<select>`, it participates in native form submission and validation out of the box. Pass `required` and it will block form submission when empty.
- **Grouping:** Use native `<optgroup label="…">` when the option count grows past ~10.
- **Accessibility:** Native `<select>` is fully accessible by default. Add `aria-label` when there's no visible label.
</Callout>
