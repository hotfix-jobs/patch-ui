# Select

A dropdown selection component with support for grouped items, separators, and multiple sizes. Custom-built on `@floating-ui/react` with `motion`-driven spring open/close, keyboard navigation, and type-ahead.

## Usage

```tsx
import {
  Select,
  SelectTrigger,
  SelectValue,
  SelectPopup,
  SelectItem,
  SelectGroup,
  SelectGroupLabel,
  SelectSeparator,
} from "@patchui/react";

// Basic select
<Select defaultValue="apple">
  <SelectTrigger>
    <SelectValue placeholder="Choose a fruit…" />
  </SelectTrigger>
  <SelectPopup>
    <SelectItem value="apple">Apple</SelectItem>
    <SelectItem value="banana">Banana</SelectItem>
    <SelectItem value="cherry">Cherry</SelectItem>
  </SelectPopup>
</Select>

// Grouped items
<Select defaultValue="react">
  <SelectTrigger className="w-56">
    <SelectValue placeholder="Choose a framework…" />
  </SelectTrigger>
  <SelectPopup>
    <SelectGroup>
      <SelectGroupLabel>Frontend</SelectGroupLabel>
      <SelectItem value="react">React</SelectItem>
      <SelectItem value="vue">Vue</SelectItem>
    </SelectGroup>
    <SelectSeparator />
    <SelectGroup>
      <SelectGroupLabel>Backend</SelectGroupLabel>
      <SelectItem value="node">Node.js</SelectItem>
    </SelectGroup>
  </SelectPopup>
</Select>
```

### SelectPopup

The popup auto-positions below the trigger with `bottom-start` placement, 4px offset, and auto-flip / shift middleware. It sizes itself to at least the trigger width and clamps its height to the available viewport (capped at 360px).

<Callout title="Notes">
- **Portal rendering:** The popup renders via a `FloatingPortal`, so it appears above all other content regardless of overflow or stacking context.
- **Composition:** Use `SelectTrigger` + `SelectValue` for the trigger. `SelectTrigger` renders the styled button with a chevron icon, and `SelectValue` displays the currently selected item text.
- **Keyboard navigation:** Supports arrow keys, type-ahead, Home/End, and Enter/Space for selection.
- **Motion:** Open and close both spring-animated (stiffness 400, damping 30, mass 0.6). Respects `prefers-reduced-motion`.
- **Auto-positioning:** Uses `flip()` + `shift()` so the popup never overflows the viewport. `autoUpdate` recalculates on scroll/resize.
- **Popup defaults:** `--radius-patch-sm` (6px), `--shadow-patch-popup`, and `--menu-item-hover` (= `--patch-accent`) for item highlight — visually consistent with Menu and Command.
</Callout>
