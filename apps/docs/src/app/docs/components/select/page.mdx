# Select

A rich single-select with a styled popup. Use `SelectItem` children; group with `SelectGroup` + `SelectGroupLabel`. For a searchable list, reach for `Combobox`.

## Usage

```tsx
import { Select, SelectItem, SelectGroup, SelectGroupLabel } from "@patchui/react";

<Select
  id="framework"
  label="Framework"
  value={framework}
  onValueChange={setFramework}
>
  <SelectItem value="react">React</SelectItem>
  <SelectItem value="vue">Vue</SelectItem>
  <SelectItem value="svelte">Svelte</SelectItem>
</Select>

// Grouped options
<Select id="assignee" label="Assignee" defaultValue="ana">
  <SelectGroup>
    <SelectGroupLabel>Design</SelectGroupLabel>
    <SelectItem value="ana">Ana</SelectItem>
    <SelectItem value="ben">Ben</SelectItem>
  </SelectGroup>
  <SelectGroup>
    <SelectGroupLabel>Engineering</SelectGroupLabel>
    <SelectItem value="chris">Chris</SelectItem>
    <SelectItem value="dana">Dana</SelectItem>
  </SelectGroup>
</Select>

// Placeholder + validation
<Select
  id="plan"
  label="Plan"
  required
  error="Please select a plan."
  placeholder="Select a plan…"
>
  <SelectItem value="hobby">Hobby</SelectItem>
  <SelectItem value="pro">Pro</SelectItem>
</Select>
```

<Callout title="Notes">
- **Rich items:** `SelectItem` accepts arbitrary children — icons, badges, two-line rows. Selection state renders a check icon via `SelectPrimitive.ItemIndicator`.
- **Grouping:** Wrap sets of items in `SelectGroup` with a `SelectGroupLabel` when the option count grows past 10.
- **Searchable list:** For 20+ items or fuzzy matching, reach for `Combobox` instead.
- **Accessibility:** Base UI Select ships proper roving tabindex, ARIA, and type-ahead out of the box.
- **Form integration:** Pass `name` and the selected value posts as a hidden input. `required` blocks submission when empty.
</Callout>
