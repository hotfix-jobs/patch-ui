# Timeline

A vertical sequence of numbered steps, editorial style. Each step is a full-width row separated by hairline borders, with the step number anchored in a left column and the title, body, optional details list, and meta line in a content column on the right. Per-step background images render subtly behind the row with a dim overlay so text stays readable.

Intended for marketing surfaces — process explanations, brand chronicles, founder's notes, "how it works" sections, any ordered narrative content. For at-a-glance summaries with 3-4 items where everything must be visible simultaneously and read in any order, prefer a 3-column grid layout instead.

## Usage

```tsx
import { Timeline } from "@patchui/react";
import { Compass, Hammer, Send } from "lucide-react";

<Timeline
  eyebrow="Process"
  heading="Three phases. One outcome."
  steps={[
    {
      n: "01",
      icon: <Compass />,
      title: "Discover",
      body: "Map the territory. Interviews, audits, and a clean read of where the product is today.",
      meta: "Roughly two weeks",
    },
    {
      n: "02",
      icon: <Hammer />,
      title: "Build",
      body: "Tight iterations against the brief. Prototypes in hand, weekly check-ins.",
      details: [
        "Design system in lockstep with engineering",
        "Reviewable diffs over slide decks",
        "Demos every Friday, no exceptions",
      ],
      meta: "Four to eight weeks, scoped",
    },
    {
      n: "03",
      icon: <Send />,
      title: "Ship",
      body: "Launch readiness check, gradual rollout, quiet handoff.",
      meta: "One week of polish and instrumentation",
    },
  ]}
/>
```

<Callout title="Notes">
- **Vertical layout:** Rows stack top-to-bottom, hairlines between. No pagination, no scroll mechanics, no nav buttons — Timeline reads like an editorial piece, not a widget.
- **Index column:** 120px on md, 160px on lg, collapses on mobile. Carries the step number and optional icon.
- **Background images:** When `bg` is set, an absolute layer renders behind the row's content with a `bg-patch-bg/55` overlay. Use sparingly — typically for one or two "hero" steps in a longer sequence.
- **Heading slot:** The `heading` prop renders via `DisplayHeading` (size `md`); pass your own JSX above the Timeline if you need a different scale or alignment.
- **SEO:** Every step renders to the DOM as a semantic `<li>` inside an ordered list. Crawlers and screen readers see the full sequence.
</Callout>
