# Badge

Short, scannable metadata that sits next to the thing it describes: status, plan tier, environment, or role. Four intent variants × high/low contrast × three sizes.

## Usage

```tsx
import { Badge } from "@patchui/react";

// Basic
<Badge>Default</Badge>

// Intent variants
<Badge variant="success">Active</Badge>
<Badge variant="warning">Pending</Badge>
<Badge variant="error">Failed</Badge>

// Contrast: high = solid saturated (default), low = subtle tint for dense surfaces
<Badge variant="success" contrast="high">Deployed</Badge>
<Badge variant="success" contrast="low">Deployed</Badge>

// Sizes
<Badge size="sm">Small</Badge>
<Badge size="md">Medium</Badge>
<Badge size="lg">Large</Badge>

// With inline icon
<Badge icon={<Shield />} variant="default">Verified</Badge>

// Shape
<Badge shape="pill">Pill</Badge>       // default
<Badge shape="rounded">Rounded</Badge> // 6px radius

// As a link
<Badge render={<a href="/status" />}>Linked</Badge>
```

<Callout title="Notes">
- Badge is display-only. If the value is clickable: remove it, open a menu, follow a link: reach for `<Button>` (with `shape="pill"` for the same silhouette).
- Keep the text tight: one word if you can, two at most (`Active`, `Pending`, `Pro`, `Enterprise Trial`). Use the same wording as the value in your API and logs so labels and data stay in sync.
- Colors carry the signal: don't double up. A `success` badge doesn't also need a check icon; an `error` badge doesn't need an ×. The color IS the meaning.
- If a row has two badges, chances are the row wanted a second column.
- `contrast="high"` is the loud version; `contrast="low"` is the quiet one for dense surfaces (tables, cards packed with metadata).
- The variant color communicates state, but color alone isn't enough: the text has to make sense on its own. For icon-only or terse badges, set `title` so screen readers can announce it.
- Don't nest a Badge inside a Badge, and don't stack two icons.
</Callout>
