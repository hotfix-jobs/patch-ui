# Badge

Short, scannable metadata that sits next to the thing it describes: status, plan tier, environment, or role. Four semantic colors × three fill variants (solid / soft / outlined) × three sizes.

## Usage

```tsx
import { Badge } from "@patchui/react";

// Basic
<Badge>Default</Badge>

// Semantic colors
<Badge color="success">Active</Badge>
<Badge color="warning">Pending</Badge>
<Badge color="error">Failed</Badge>

// Fill variants: solid = saturated, soft (default) = tinted, outlined = hairline border
<Badge color="success" variant="solid">Deployed</Badge>
<Badge color="success" variant="soft">Deployed</Badge>
<Badge color="success" variant="outlined">Deployed</Badge>

// Sizes
<Badge size="sm">Small</Badge>
<Badge size="md">Medium</Badge>
<Badge size="lg">Large</Badge>

// With inline icon
<Badge icon={<Shield />} color="default">Verified</Badge>

// Shape
<Badge shape="pill">Pill</Badge>       // default
<Badge shape="rounded">Rounded</Badge> // 6px radius

// As a link
<Badge render={<a href="/status" />}>Linked</Badge>
```

<Callout title="Notes">
- Badge is display-only. If the value is clickable: remove it, open a menu, follow a link: reach for `<Button>` (with `shape="pill"` for the same silhouette).
- Keep the text tight: one word if you can, two at most (`Active`, `Pending`, `Pro`, `Enterprise Trial`). Use the same wording as the value in your API and logs so labels and data stay in sync.
- Colors carry the signal: don't double up. A `success` badge doesn't also need a check icon; an `error` badge doesn't need an ×. The color IS the meaning.
- If a row has two badges, chances are the row wanted a second column.
- `variant="solid"` is the loud version; `variant="soft"` is the quiet one for dense surfaces (tables, cards packed with metadata); `variant="outlined"` reads as a data chip when a fill would compete with a neighbor.
- Color communicates state, but color alone isn't enough: the text has to make sense on its own. For icon-only or terse badges, set `title` so screen readers can announce it.
- Don't nest a Badge inside a Badge, and don't stack two icons.
</Callout>
