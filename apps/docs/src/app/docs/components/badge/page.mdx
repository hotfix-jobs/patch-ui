# Badge

Short, scannable metadata that sits next to the thing it describes: status, plan tier, environment, or role. Four intent variants × high/low contrast × three sizes.

## Usage

```tsx
import { Badge } from "@patchui/react";

// Basic
<Badge>Default</Badge>

// Intent variants
<Badge variant="success">Active</Badge>
<Badge variant="warning">Pending</Badge>
<Badge variant="error">Failed</Badge>

// Contrast: high = solid saturated (default), low = subtle tint for dense surfaces
<Badge variant="success" contrast="high">Deployed</Badge>
<Badge variant="success" contrast="low">Deployed</Badge>

// Sizes
<Badge size="sm">Small</Badge>
<Badge size="md">Medium</Badge>
<Badge size="lg">Large</Badge>

// With inline icon
<Badge icon={<Shield />} variant="default">Verified</Badge>

// Shape
<Badge shape="pill">Pill</Badge>       // default
<Badge shape="rounded">Rounded</Badge> // 6px radius

// As a link
<Badge render={<a href="/status" />}>Linked</Badge>
```

<Callout title="Content and usage">
- **Static labels**: badges don't take `onClick`. If the user can act on the value, promote to a `<Button>` or `<Pill>`.
- **Content**: Title Case, one word when possible: `Active`, `Pending`, `Pro`, `Enterprise Trial`. Match the canonical API term (`Production`, not `Prod`).
- **Don't stack signals**: no checkmark icon for success, no × for error — the variant carries the meaning.
- **One per row**: two badges side by side means the row needs a second column.
- **Meaning maps to color**: green = healthy, red = error, amber = warning, blue = info, gray = neutral. `contrast="low"` tones any of them down for dense surfaces.
- **A11y**: set `title` on icon-only or ambiguous badges so screen readers announce the meaning.
- **Never nest badges** or stack icons inside a badge.
</Callout>
