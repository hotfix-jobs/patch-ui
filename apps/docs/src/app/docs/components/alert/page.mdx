# Alert

Inline banner for info / success / warning / danger messages. Hairline border, tinted background per variant, optional icon, title, description, and dismiss × button. Fades in on mount, slides out smoothly when dismissed.

## Usage

```tsx
import { Alert } from "@patchui/react";

<Alert title="Heads up">Your changes are saved every 30 seconds.</Alert>

<Alert variant="success" title="Saved">
  All your work has been committed.
</Alert>

<Alert variant="warning" title="Storage almost full">
  You're using 92% of your plan.
</Alert>

<Alert variant="danger" title="Could not connect" dismissible>
  We couldn't reach the server.
</Alert>

// Override the default icon
<Alert variant="info" icon={<Mail className="size-4" />} title="New mail">
  3 unread messages.
</Alert>

// Hide the icon
<Alert variant="info" icon={null} title="Quiet alert">
  No icon shown.
</Alert>
```

<Callout title="Notes">
- **Motion:** Fades in with a subtle slide on mount, exits smoothly when dismissed via `AnimatePresence`. Respects `prefers-reduced-motion`.
- **Tinted background:** Uses `color-mix(in oklab, var(--badge-*-text) 8%, var(--background-100))` so the variant color subtly tints the surface without overwhelming the content. Border is the same color at 30% opacity.
- **Icon color:** The default icon inherits the variant's color token (`--badge-success-text`, etc.). The body text stays neutral so the message reads cleanly.
</Callout>
