# Slider

A slider for selecting a single value or a range, with built-in track, thumb rendering, and an optional value display.

<Install name="slider" />

<ComponentPreview label="Slider Examples">
  <SliderDemo />
</ComponentPreview>

## Usage

```tsx
import { Slider, SliderValue } from "@patchui/react";

// Single value
<Slider defaultValue={50} />

// With value display
<Slider defaultValue={50}>
  <SliderValue />
</Slider>

// Range (two thumbs)
<Slider defaultValue={[20, 80]} />

// Custom step
<Slider defaultValue={50} step={10} min={0} max={100} />

// Controlled
const [value, setValue] = useState(30);
<Slider value={value} onValueChange={(v) => setValue(v)} />

// Range with numeric inputs on each end (range mode only)
const [range, setRange] = useState([50, 75]);
<Slider
  value={range}
  onValueChange={setRange}
  showStartInput
  showEndInput
/>
```

<Callout title="Notes">
- Reach for a slider when the shape of the value matters more than precision — bandwidth caps, opacity, audio volume, color channels. For exact values (port numbers, memory limits), pair the slider with a numeric `<Input>` so keyboard users can type.
- Pass a sensible `step` (`1`, `5`, `10%`) so dragging never produces `47.83291`. Clamp `min` and `max` to real product limits.
- Always render the live value next to the track and label what the number represents (`Sample Rate · 44 kHz`). The track alone doesn't say what's being adjusted.
- Range mode: pass an array to `defaultValue` or `value` (`[20, 80]`) and the slider renders one thumb per value.
- `onValueChange` fires continuously — good for live preview. `onValueCommitted` fires on drag-end — good for saving to a database.
- The native arrow keys, Page Up/Down, and Home/End move the thumb — don't intercept those key handlers.
</Callout>
