# Avatar

A user avatar built on Base UI's Avatar. Shows an image, and gracefully falls back to a solid monogram (initials, or any node) while the image loads or if it fails. Five sizes from `xs` to `xl`, `circle` or `square` shape. The fallback is a high-contrast monogram (dark fill + light initials) that inverts automatically in dark mode.

Compose multiple avatars into an overlapping row with `AvatarGroup` — handles the size/shape propagation, hairline ring against the background, and overflow chip (`+N`) automatically.

## Usage

```tsx
import {
  Avatar,
  AvatarImage,
  AvatarFallback,
  AvatarGroup,
} from "@patchui/react";

<Avatar>
  <AvatarImage src="/people/jane.jpg" alt="Jane Doe" />
  <AvatarFallback>JD</AvatarFallback>
</Avatar>

// Sizes
<Avatar size="xs" />  // 24px — tight inline contexts
<Avatar size="sm" />  // 28px
<Avatar size="md" />  // 36px (default)
<Avatar size="lg" />  // 44px
<Avatar size="xl" />  // 56px — profile heroes

// Square
<Avatar shape="square">
  <AvatarImage src="/logo.png" alt="Acme" />
  <AvatarFallback>A</AvatarFallback>
</Avatar>

// Overlapping stack with overflow chip
<AvatarGroup size="sm" max={4}>
  <Avatar><AvatarImage src="/a.png" /></Avatar>
  <Avatar><AvatarFallback>JD</AvatarFallback></Avatar>
  <Avatar><AvatarFallback>SR</AvatarFallback></Avatar>
  <Avatar><AvatarFallback>MK</AvatarFallback></Avatar>
  <Avatar><AvatarFallback>LF</AvatarFallback></Avatar>
  <Avatar><AvatarFallback>BV</AvatarFallback></Avatar>
  {/* 6 children, max=4 → renders 4 avatars + "+2" chip */}
</AvatarGroup>
```

## Components

- `Avatar` — root container with size/shape variants and a hairline ring
- `AvatarImage` — the image; covers the container and hides itself if it fails to load
- `AvatarFallback` — shown while the image loads or on error; typically initials
- `AvatarGroup` — overlapping row of avatars with auto-applied size/shape + ring + `+N` overflow chip

<Callout title="Notes">
- **Graceful fallback:** `AvatarFallback` renders until the image successfully loads, so there's no broken-image flash. Use 1-2 character initials, an icon, or any node.
- **AvatarGroup composition:** Children are expected to be `<Avatar>` elements. AvatarGroup propagates size/shape, adds a `ring-background-100` hairline between avatars so they visually separate despite overlapping, and tightens negative margins based on size.
- **Overflow chip:** When more children are passed than `max`, the remaining count is rendered as a `+N` chip with a `--patch-surface-2` fill — distinct from the dark Avatar fill so it reads as a counter rather than another person.
- **xl + heroes:** Use `xl` (56px) for profile headers, agent / persona cards, and big-display moments. For inline mentions, `xs` is right.
</Callout>
