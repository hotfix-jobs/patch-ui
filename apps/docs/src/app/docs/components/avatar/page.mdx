# Avatar

A user or team avatar. Built on Base UI's Avatar; falls back gracefully from image → monogram. Sizes accept enum (`xs`–`xl`) or an exact pixel number. `AvatarGroup` takes either a `members` array or nested `<Avatar>` children.

## Usage

```tsx
import {
  Avatar,
  AvatarImage,
  AvatarFallback,
  AvatarGroup,
} from "@patchui/react";

// Single avatar
<Avatar>
  <AvatarImage src="/people/jane.jpg" alt="Jane Doe" />
  <AvatarFallback>JD</AvatarFallback>
</Avatar>

// Sizes: enum
<Avatar size="xs" />  // 24px
<Avatar size="sm" />  // 28px
<Avatar size="md" />  // 36px (default)
<Avatar size="lg" />  // 44px
<Avatar size="xl" />  // 56px

// Sizes: numeric (any pixel)
<Avatar size={32} />
<Avatar size={64} />

// Letter shorthand + placeholder fill
<Avatar letter="SL" placeholder size={32} />

// Placeholder person icon (no image, no letter)
<Avatar placeholder size={90} />

// Square
<Avatar shape="square" size={40}>
  <AvatarImage src="/logo.png" alt="Acme" />
  <AvatarFallback>A</AvatarFallback>
</Avatar>

// Group with a members array
<AvatarGroup
  size={32}
  members={[
    { image: "/people/a.jpg", letter: "AB" },
    { image: "/people/b.jpg", letter: "CD" },
    { image: "/people/c.jpg", letter: "EF" },
  ]}
/>

// Overflow chip
<AvatarGroup size={32} limit={4} members={sixMembers} />

// Overlap: auto scales with size; number = exact pixel overlap
<AvatarGroup size={24} overlap="auto" members={m} />
<AvatarGroup size={24} overlap={0} members={m} />

// Reverse stacking (last member on top)
<AvatarGroup size={32} reverse members={m} />
```

## Components

- `Avatar` — the container
- `AvatarImage` — the image; covers the container and hides itself on error
- `AvatarFallback` — shown while the image loads or on error; typically initials
- `AvatarGroup` — overlapping row with data-driven or children API, auto-scaling overlap, and `+N` overflow chip

<Callout title="Notes">
- Both size APIs coexist. Reach for the enum when a preset lines up (`sm`, `md`, `lg`…). Reach for a number when the layout needs an exact pixel diameter.
- Inside a group, each avatar picks up a `ring-background-100` hairline. That's what keeps overlapping avatars from bleeding into each other.
- The overflow chip uses `bg-gray-100` + `text-gray-900` — visibly different from a real avatar, reads as a counter.
- Pick the `members` prop for lists coming from data, keep children when individual avatars need their own tooltip, click handler, or fallback.
- `reverse` flips the z-stack. Left-to-right visual order stays the same; what changes is who sits on top when they overlap.
- The `icon` prop is a small status badge tucked into the bottom-left corner, not a replacement for the avatar body. The main avatar (image/letter/placeholder) still renders behind it.
</Callout>
