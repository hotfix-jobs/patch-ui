# ThemeToggle

A light/dark mode toggle button with animated sun/moon icons. Works uncontrolled with localStorage persistence, or controlled via props. Applies the `.dark` class to `<html>` by default.

## Usage

```tsx
import { ThemeToggle } from "@patchui/react";

// Uncontrolled - persists to localStorage, applies .dark class
<ThemeToggle />

// Controlled
const [theme, setTheme] = useState<"light" | "dark">("light");
<ThemeToggle
  theme={theme}
  onThemeChange={setTheme}
/>

// With next-themes
import { useTheme } from "next-themes";

function MyToggle() {
  const { resolvedTheme, setTheme } = useTheme();
  return (
    <ThemeToggle
      theme={resolvedTheme as "light" | "dark"}
      onThemeChange={setTheme}
      applyClass={false}
      storageKey={false}
    />
  );
}
```

## Flash Prevention

Add this inline script to your `<head>` to prevent a flash of wrong theme on page load:

```html
<script>
  (function() {
    var t = localStorage.getItem('patch-ui-theme');
    if (!t) t = matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
    if (t === 'dark') document.documentElement.classList.add('dark');
  })();
</script>
```

<Callout title="Notes">
- Uncontrolled mode is the common path: ThemeToggle reads from localStorage on mount, applies the `.dark` class to `<html>`, and persists on every toggle. No provider needed.
- For controlled mode (usually alongside `next-themes`), pass `theme` and `onThemeChange`, then set `applyClass={false}` and `storageKey={false}` so ThemeToggle doesn't double-write.
- SSR-safe via `useSyncExternalStore`: the button renders a blank placeholder before hydration so the first client render matches the server.
- The sun/moon icons crossfade with a spring rotation on toggle. Reduced motion swaps them instantly.
- The icon-only control inherits Button's default rounded shape so it aligns with header and toolbar utilities.
- If you need a 3-option picker (Light / System / Dark) with System as a first-class option, compose one with `ToggleGroup` and three icons.
</Callout>
