# Input

A styled text input with three visual variants (outlined, ghost, underline), sizes, icons, prefix/suffix slots, error and loading states, a clear button, and an unstyled escape hatch.

## Usage

```tsx
import { Input } from "@patchui/react";
import { Search } from "lucide-react";

// Basic (outlined is the default)
<Input placeholder="Enter text..." />

// Variants
<Input variant="outlined" placeholder="Outlined" />
<Input variant="ghost" placeholder="Ghost" />          // no border / bg, embedded
<Input variant="underline" placeholder="Underline" />  // bottom border only

// With icon
<Input icon={<Search className="h-4 w-4" />} placeholder="Search..." />

// With prefix and suffix
<Input prefix="$" placeholder="0.00" />
<Input suffix=".com" placeholder="username" />

// Sizes
<Input size="sm" placeholder="Small" />
<Input size="lg" placeholder="Large" />

// Error state
<Input invalid value={email} onChange={...} />

// Loading state — disables input, shows spinner
<Input loading value={value} />

// Clear button — appears on the right when value is non-empty
<Input
  value={query}
  onChange={(e) => setQuery(e.target.value)}
  onClear={() => setQuery("")}
/>

// Unstyled (bring your own styles)
<Input unstyled className="my-custom-input" />
```

<Callout title="Notes">
- **Variant + size compose:** every variant accepts `sm` / `md` / `lg` — for example `<Input variant="underline" size="lg" />` is the search input used inside `CommandDialog`.
- **`icon` vs `prefix`:** both render at the start of the input. Use `icon` for SVG icons and `prefix` for text-based prefixes like `$` or `@`.
- **`onClear` ergonomics:** the × button auto-hides while the input is empty, disabled, or loading. It also sets `tabIndex={-1}` so Tab doesn't stop on it.
- **Loading overrides suffix:** while `loading` is true, the spinner replaces any user-provided `suffix` to avoid layout collisions.
- **Unstyled mode:** still renders the wrapper `<span>` and slot structure so you can layer your own classes on top.
- **Validation messaging:** for description and error text *below* the input, use the `Field` component which wires `aria-describedby` to your message.
</Callout>
