# Input

A text input with three sizes, icon and prefix/suffix slots, and built-in error, loading, and clear states. Follows the single input pattern from the design bar: `background-100` fill, translucent border, 6px radius.

## Usage

```tsx
import { Input } from "@patchui/react";
import { Search } from "lucide-react";

// Basic
<Input placeholder="Enter text" />

// Sizes
<Input size="sm" placeholder="Small" />   // 32px
<Input size="md" placeholder="Medium" />  // 40px (default)
<Input size="lg" placeholder="Large" />   // 48px

// With icon
<Input icon={<Search className="h-4 w-4" />} placeholder="Search" />

// Prefix and suffix
<Input prefix="$" placeholder="0.00" />
<Input suffix=".com" placeholder="username" />

// Error state
<Input invalid value={email} onChange={…} />

// Loading — disables input, shows spinner in trailing slot
<Input loading value={value} />

// Clear button — appears when value is non-empty
<Input
  value={query}
  onChange={(e) => setQuery(e.target.value)}
  onClear={() => setQuery("")}
/>
```

<Callout title="Notes">
- **Single pattern:** one style — `background-100` fill, `gray-alpha-400` border stepping to `-500` on hover and `-600` on focus, 6px radius. No variant sprawl.
- **`icon` vs `prefix`:** both render at the start. Use `icon` for SVG icons and `prefix` for text prefixes like `$` or `@`.
- **`onClear` ergonomics:** the × auto-hides while empty, disabled, or loading. `tabIndex={-1}` so Tab doesn't stop on it.
- **Loading overrides suffix:** while `loading` is true, the spinner replaces any user-provided `suffix` to avoid layout collisions.
- **Validation messaging:** for description and error text below the input, use `Field` which wires `aria-describedby` to your message.
</Callout>
