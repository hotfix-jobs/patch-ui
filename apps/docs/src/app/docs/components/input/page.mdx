# Input

A focused text-control primitive with three sizes and optional `prefix` and `suffix` slots. It uses a neutral fill, no resting border, radius 8, and a crisp solid focus outline. Compose labels, descriptions, and errors with `Field`.

## Usage

```tsx
import { Field, FieldError, FieldLabel, Input } from "@patchui/react";
import { MagnifyingGlass, Envelope, ArrowCircleUp } from "@phosphor-icons/react/dist/ssr";
// Basic
<Input placeholder="Enter text" />

// Sizes
<Input size="sm" placeholder="Small" />   // 32px
<Input size="md" placeholder="Medium" />  // 40px (default)
<Input size="lg" placeholder="Large" />   // 48px

// Prefix and suffix (inline muted glyph or short label)
<Input prefix={<ArrowUpCircle />} placeholder="Icon prefix" />
<Input suffix={<ArrowUpCircle />} placeholder="Icon suffix" />
<Input prefix="https://" suffix=".com" placeholder="Text prefix and suffix" />

// Label and error composition
<Field invalid>
  <FieldLabel>Email</FieldLabel>
  <Input id="email" type="email" />
  <FieldError match>Enter a valid email address.</FieldError>
</Field>

// Loading: disables input, shows a spinner
<Input loading value={value} />

// For clearable search, use SearchInput (X button clears the value)
<SearchInput
  value={query}
  onChange={(e) => setQuery(e.target.value)}
  placeholder="Search projects"
/>
```

<Callout title="Notes">
- Compose label, description, invalid state, and error messaging with `Field`.
- Whitespace-trim the value at submit time so a stray space doesn't slip past a required check.
- Placeholders should look like a real value the user would type: `example.com`, `my-project`. Skip the "Enter your…" instructions; the label already does that job.
- Prefix and suffix render inline in muted color. Good for icons (`<Search />`, `<Mail />`), short text (`https://`, `.com`, currency symbols), and Kbd hints (`Esc`).
- For an action button that sits inside an input row, pass an icon-only Button with `aria-label`, not a naked icon.
- Use `<SearchInput>` for search boxes: the trailing X button appears once the field has a value and clears it on click.
</Callout>
