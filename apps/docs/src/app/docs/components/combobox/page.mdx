# Combobox

An `Input` paired with a floating popup that shows arbitrary filterable content: suggestions, recent searches, custom rows, banners. Built on `@floating-ui/react` with virtual focus (the input keeps real focus; items highlight via `aria-activedescendant`) and motion-driven open / close.

Use Combobox when you need an input-driven dropdown with custom row shapes (recents with chips, sign-in nudges, mode-specific banners). For simple command palettes, use [Command](/docs/components/command). For trigger-button dropdowns, use [Menu](/docs/components/menu).

Query state (`value` / `onValueChange` / `placeholder`) lives on the `<Combobox>` root so the trigger input and the mobile panel's own search input can share it. On viewports below the `md` breakpoint the popup automatically switches to a bottom-anchored panel with its own search input.

## Usage

```tsx
import {
  Combobox,
  ComboboxInput,
  ComboboxPopup,
  ComboboxItem,
} from "@patchui/react";
import { MagnifyingGlass } from "@phosphor-icons/react";
const [query, setQuery] = useState("");
const matches = items.filter((i) =>
  i.label.toLowerCase().includes(query.toLowerCase()),
);

<Combobox value={query} onValueChange={setQuery} placeholder="Search...">
  <ComboboxInput prefix={<Search />} />
  <ComboboxPopup>
    {matches.map((item) => (
      <ComboboxItem key={item.id} onSelect={() => pick(item)}>
        {item.label}
      </ComboboxItem>
    ))}
  </ComboboxPopup>
</Combobox>
```

For multi-select, pass `closeOnSelect={false}` on `ComboboxItem` so the popup stays open while the user toggles rows:

```tsx
<Combobox value={query} onValueChange={setQuery}>
  <ComboboxInput prefix={<Search />} />
  <ComboboxPopup>
    {matches.map((item) => (
      <ComboboxItem
        key={item.id}
        closeOnSelect={false}
        onSelect={() => toggle(item)}
      >
        {item.label} {selected.has(item.id) && <Check />}
      </ComboboxItem>
    ))}
  </ComboboxPopup>
</Combobox>
```

The popup accepts arbitrary JSX: only `ComboboxItem` gets keyboard navigation. Mix items with banners, headers, separators, or custom rows freely.

```tsx
<ComboboxPopup>
  {/* Info banner — not navigable */}
  <div className="flex items-center gap-1.5 border-b-[0.5px] border-hairline-strong px-3 py-2 text-mini text-ink-muted">
    <Info className="h-3.5 w-3.5" /> Recents shown first
  </div>

  {/* Recents section (use ComboboxItem so they're keyboard-navigable) */}
  {recents.map((r) => (
    <ComboboxItem key={r.id} onSelect={() => useRecent(r)}>
      <RecentRow search={r} />
    </ComboboxItem>
  ))}

  {/* Live suggestions */}
  {suggestions.map((s) => (
    <ComboboxItem key={s} onSelect={() => pick(s)}>
      {s}
    </ComboboxItem>
  ))}
</ComboboxPopup>
```

<Callout title="Notes">
- **Query state on the root.** Put `value` / `onValueChange` / `placeholder` on `<Combobox>` so the trigger input and the mobile panel's own input share them. The input still accepts `value` / `onChange` / `placeholder` as per-input overrides for advanced cases.
- **Mobile panel.** Below 768px the popup renders as a bottom-anchored fixed panel with its own search input (real focus, autoFocus) and a close X. Desktop behavior is unchanged.
- **Multi-select.** Pass `closeOnSelect={false}` on items you want to toggle without dismissing the popup. Track selection in your own state.
- The input keeps real focus while you type and arrow through options on desktop. Items get `aria-activedescendant` instead of taking focus, which is the proper ARIA combobox pattern: screen readers announce the active option without interrupting typing.
- Combobox doesn't filter for you. Filter your data based on the current input value and render the matches as `<ComboboxItem>` children. Substring, fuzzy index, async API: any strategy works.
- Anything inside `ComboboxPopup` renders. Only `<ComboboxItem>` is keyboard-navigable, so you can drop in banners, section headers, and dividers as plain JSX between item groups.
- Keyboard: Arrow keys navigate, Enter selects, Esc closes, ↓ opens when closed. Typing is just text input: the popup stays open.
- The chevron suffix auto-flips based on open state. Use `hideChevron` when the input is pure search (no dropdown affordance needed).
- `clearable` renders a trailing × when the input has a value. Reset your `value` state in `onClear`: the component doesn't own the value.
- Motion: spring-physics open / close. Respects `prefers-reduced-motion`.
- Anchored 8px below the input, flips above when there's no room, shifts to stay 8px from edges. Popup `minWidth` matches the input width.
</Callout>
