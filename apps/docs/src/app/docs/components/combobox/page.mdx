# Combobox

An `Input` paired with a floating popup that shows arbitrary filterable content: suggestions, recent searches, custom rows, banners. Built on Base UI Combobox: virtual focus (the input keeps real focus; items highlight via `aria-activedescendant`), roving keyboard navigation, and CSS-driven open / close transitions.

Use Combobox when you need an input-driven dropdown with custom row shapes (recents with chips, sign-in nudges, mode-specific banners). For simple command palettes, use [Command](/docs/components/command). For trigger-button dropdowns, use [Menu](/docs/components/menu).

Query state (`value` / `onValueChange` / `placeholder`) lives on the `<Combobox>` root so the trigger input and the mobile panel's own search input can share it. On viewports below the `md` breakpoint the popup automatically switches to a bottom-anchored panel with its own search input.

## Usage

```tsx
import {
  Combobox,
  ComboboxInput,
  ComboboxPopup,
  ComboboxItem,
} from "@patchui/react";
import { MagnifyingGlass } from "@phosphor-icons/react/dist/ssr";
const [query, setQuery] = useState("");
const matches = items.filter((i) =>
  i.label.toLowerCase().includes(query.toLowerCase()),
);

<Combobox value={query} onValueChange={setQuery} placeholder="Search...">
  <ComboboxInput prefix={<Search />} />
  <ComboboxPopup>
    {matches.map((item) => (
      <ComboboxItem key={item.id} onSelect={() => pick(item)}>
        {item.label}
      </ComboboxItem>
    ))}
  </ComboboxPopup>
</Combobox>
```

### Multi-select

Pass `multiple` on the root with `selectedValues` + `onSelectedValuesChange` and Base UI manages selection state internally. Clicking a `ComboboxItem` toggles it in the array; the built-in `clearable` X on `ComboboxInput` becomes visible whenever `selectedValues` is non-empty and wipes both the input and the picks on click.

```tsx
const [query, setQuery] = useState("");
const [selected, setSelected] = useState<string[]>([]);

<Combobox
  multiple
  selectedValues={selected}
  onSelectedValuesChange={(next) => setSelected(next as string[])}
  value={query}
  onValueChange={setQuery}
  placeholder="Frameworks"
>
  <ComboboxInput
    prefix={<Search />}
    clearable
    onClear={() => {
      setQuery("");
      setSelected([]);
    }}
  />
  <ComboboxPopup>
    {matches.map((item) => (
      <ComboboxItem key={item.id} value={item.id}>
        {item.label}
      </ComboboxItem>
    ))}
  </ComboboxPopup>
</Combobox>
```

To render a Selected section at the top of the popup so users can see and remove active picks in place, pass `onRemove` on the `ComboboxItem` for each row:

```tsx
<ComboboxPopup>
  {query.trim() === "" && selected.length > 0 ? (
    <>
      <div className="px-2 pb-1 pt-2 text-mini text-ink-muted">Selected</div>
      {selected.map((s) => (
        <ComboboxItem
          key={`sel-${s}`}
          value={s}
          onRemove={() => setSelected((prev) => prev.filter((x) => x !== s))}
          removeLabel={`Remove ${s}`}
        >
          {s}
        </ComboboxItem>
      ))}
    </>
  ) : (
    matches.map((item) => (
      <ComboboxItem key={item.id} value={item.id}>
        {item.label}
      </ComboboxItem>
    ))
  )}
</ComboboxPopup>
```

To display a summary of the picks in the input when the popup is closed, thread through an `open` state and compute the display value:

```tsx
const [open, setOpen] = useState(false);
const summary = selected.length === 1 ? selected[0] : `${selected.length} frameworks`;
const displayValue = open ? query : query || summary;

<Combobox
  multiple
  selectedValues={selected}
  onSelectedValuesChange={...}
  open={open}
  onOpenChange={setOpen}
  value={displayValue}
  onValueChange={(v) => {
    // Ignore the summary being echoed back on close.
    if (!open && v === summary) return;
    setQuery(v);
  }}
>
  ...
</Combobox>
```

When the popup opens the input clears to a raw search field so typing / backspacing behaves normally; when it closes the summary comes back as a display-only status.

The popup accepts arbitrary JSX: only `ComboboxItem` gets keyboard navigation. Mix items with banners, headers, separators, or custom rows freely.

```tsx
<ComboboxPopup>
  {/* Info banner — not navigable */}
  <div className="flex items-center gap-1.5 border-b-[0.5px] border-hairline-strong px-3 py-2 text-mini text-ink-muted">
    <Info className="h-3.5 w-3.5" /> Recents shown first
  </div>

  {/* Recents section (use ComboboxItem so they're keyboard-navigable) */}
  {recents.map((r) => (
    <ComboboxItem key={r.id} onSelect={() => useRecent(r)}>
      <RecentRow search={r} />
    </ComboboxItem>
  ))}

  {/* Live suggestions */}
  {suggestions.map((s) => (
    <ComboboxItem key={s} onSelect={() => pick(s)}>
      {s}
    </ComboboxItem>
  ))}
</ComboboxPopup>
```

<Callout title="Notes">
- **Query state on the root.** Put `value` / `onValueChange` / `placeholder` on `<Combobox>` so the trigger input and the mobile panel's own input share them. The input still accepts `value` / `onChange` / `placeholder` as per-input overrides for advanced cases.
- **Mobile panel.** Below 768px the popup renders as a bottom-anchored fixed panel with its own search input (real focus, autoFocus) and a close X. Desktop behavior is unchanged.
- **Multi-select.** Pass `multiple` on the root with `selectedValues` + `onSelectedValuesChange` and Base UI manages the array natively (the built-in clearable X becomes visible when the array is non-empty and clears both the input and the picks on click). For the "list of active picks with individual X to remove" pattern, put `onRemove` on the `ComboboxItem` for each Selected row.
- **State consistency.** The open input uses the same stronger fill as Select and Menu triggers, while retaining its editable-field focus outline. Highlighted rows are subtle; selected rows keep the stronger selected fill.
- The input keeps real focus while you type and arrow through options on desktop. Items get `aria-activedescendant` instead of taking focus, which is the proper ARIA combobox pattern: screen readers announce the active option without interrupting typing.
- Combobox doesn't filter for you. Filter your data based on the current input value and render the matches as `<ComboboxItem>` children. Substring, fuzzy index, async API: any strategy works.
- Anything inside `ComboboxPopup` renders. Only `<ComboboxItem>` is keyboard-navigable, so you can drop in banners, section headers, and dividers as plain JSX between item groups.
- Keyboard: Arrow keys navigate, Enter selects, Esc closes, ↓ opens when closed. Typing is just text input: the popup stays open.
- The chevron suffix auto-flips based on open state. Use `hideChevron` when the input is pure search (no dropdown affordance needed).
- `clearable` renders a trailing × when the input has a value. Reset your `value` state in `onClear`: the component doesn't own the value.
- Motion: spring-physics open / close. Respects `prefers-reduced-motion`.
- Anchored 8px below the input, flips above when there's no room, shifts to stay 8px from edges. Popup `minWidth` matches the input width.
</Callout>
