# Sidebar

A compound sidebar primitive for docs and app layouts. `SidebarProvider` owns open/collapsed state and wires a `Cmd+B` / `Ctrl+B` keyboard shortcut by default. On desktop the sidebar is a fixed-position column with a spacer that reserves width in flex flow; on mobile it becomes a `Sheet`-based drawer with focus trap, escape-to-close, and portal rendering.

The desktop surface is borderless `layer-1`. Navigation items use the same hover, selected, and compact keyboard-focus states as Tabs.

Inspired by shadcn/ui's Sidebar block. Ships the same compound API: `SidebarProvider`, `Sidebar`, `SidebarInset`, `SidebarHeader`, `SidebarContent`, `SidebarFooter`, `SidebarGroup`, `SidebarGroupLabel`, `SidebarMenu`, `SidebarMenuItem`, `SidebarMenuButton`, `SidebarTrigger`, and the `useSidebar` hook.

## Usage

```tsx
import {
  Sidebar,
  SidebarContent,
  SidebarGroup,
  SidebarGroupLabel,
  SidebarHeader,
  SidebarInset,
  SidebarMenu,
  SidebarMenuButton,
  SidebarMenuItem,
  SidebarProvider,
  SidebarTrigger,
} from "@patchui/react";

<SidebarProvider defaultOpen>
  <Sidebar rounded>
    <SidebarHeader>{/* brand */}</SidebarHeader>
    <SidebarContent>
      <SidebarGroup>
        <SidebarGroupLabel>Workspace</SidebarGroupLabel>
        <SidebarMenu>
          <SidebarMenuItem>
            <SidebarMenuButton active render={<Link href="/" />}>
              Overview
            </SidebarMenuButton>
          </SidebarMenuItem>
        </SidebarMenu>
      </SidebarGroup>
    </SidebarContent>
  </Sidebar>
  <SidebarInset>
    <header>
      <SidebarTrigger />
    </header>
    <main>{children}</main>
  </SidebarInset>
</SidebarProvider>
```

Use `rounded` when the sidebar is presented as an inset surface. Leave it off for a flush application edge.

<Callout title="Notes">
- **Cmd+B shortcut** ships wired by default. Pass `shortcut="mod+i"` to change the key or `shortcut={null}` to disable.
- **Mobile drawer** delegates to `Sheet`, so it inherits focus trap, escape-to-close, and portal rendering. No manual backdrop wiring.
- **SidebarInset** is the main content column. It sits alongside the sidebar via a shared flex row provided by `SidebarProvider`. If you use Sidebar without SidebarInset, make sure the sibling main element is a flex-1 block.
- **Active state** is set with the `active` prop on `SidebarMenuButton`. In routing frameworks, compute active from `pathname === item.href` and pass through.
</Callout>
