# Section

A structural card surface with optional header, content, and footer slots. It is borderless and shadowless by default. Add `dividers` when direct children need quiet internal separation.

Different from `Card`, which represents a discrete content object. Reach for Section when several controls or settings belong to one named area.

## Usage

```tsx
import {
  Section,
  SectionHeader,
  SectionTitle,
  SectionSubtitle,
  SectionContent,
  SectionFooter,
  Button,
  Input,
} from "@patchui/react";

// Simple settings row (header + footer, no body)
<Section dividers>
  <SectionHeader className="px-6 pb-4 pt-5">
    <SectionTitle>Account Settings</SectionTitle>
    <SectionSubtitle>Manage your account preferences and settings.</SectionSubtitle>
  </SectionHeader>
  <SectionFooter className="flex items-center justify-between px-6 pb-5">
    <div className="text-small text-ink-muted">Need help? <a href="#">View documentation</a></div>
    <div className="flex items-center gap-2">
      <Button size="sm">Save Changes</Button>
    </div>
  </SectionFooter>
</Section>

// With a content body (e.g. an input to edit)
<Section dividers>
  <SectionHeader className="px-6 pb-4 pt-5">
    <SectionTitle>Project Name</SectionTitle>
    <SectionSubtitle>The display name for your project.</SectionSubtitle>
  </SectionHeader>
  <SectionContent className="px-6 pb-6">
    <Input placeholder="my-awesome-project" />
  </SectionContent>
  <SectionFooter className="flex items-center justify-between px-6 pb-5">
    <div className="text-small text-ink-muted">Please use 32 characters at maximum.</div>
    <div className="flex items-center gap-2">
      <Button size="sm">Save</Button>
    </div>
  </SectionFooter>
</Section>
```

## Components

- **`Section`**: the structural surface. Renders as `<section>` by default.
- **`SectionHeader`**: the top block: title + subtitle.
- **`SectionTitle`**: heading (renders as `<h3>` for correct outline).
- **`SectionSubtitle`**: descriptive text under the title (renders as `<p>`).
- **`SectionContent`**: optional body between header and footer.
- **`SectionFooter`**: optional unstyled footer slot.

### SectionHeader, SectionTitle, SectionSubtitle, SectionContent, SectionFooter

Each accepts `className` for overrides and `render` for polymorphic rendering (change the underlying element while keeping the styling).

<Callout title="Notes">
- Section slots are unstyled structural hooks. Callers own their padding, gaps, alignment, and responsive behavior through `className`.
- Section is a `<section>` element; `SectionTitle` renders as `<h3>` by default. If the parent already has an `<h2>`, that keeps the outline clean. Adjust with `render={<h2 />}` if the heading level needs to shift.
- Section has one card treatment: `bg-layer-1`, radius 12, no outer border, and no shadow. Use Card when an object needs outlined or elevated variants.
- `dividers` adds internal hairlines between direct children. Leave it off when spacing alone makes the grouping clear.
</Callout>
