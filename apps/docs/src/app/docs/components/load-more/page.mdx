# LoadMore

A centered secondary-style button that appends more items to a paginated feed: job listings, search results, activity logs, anywhere the user wants to reveal more of the same data set without changing pages. Renders as a hug-width pill (`w-fit` + `mx-auto`) so it reads as a discrete action, not a full-width footer bar.

For sequential navigation between distinct pages, use `Pagination`.

## Usage

```tsx
import { LoadMore } from "@patchui/react";

// Basic
<LoadMore onClick={loadMore}>Load More</LoadMore>

// Loading state
<LoadMore loading>Loading…</LoadMore>

// Attached to a list container — no top margin, flush corners
<Card border borderBetween>
  {items.map(row => <Row key={row.id} row={row} />)}
  <LoadMore noGap noBorderRadius loading={fetching} onClick={loadMore}>
    Load More
  </LoadMore>
</Card>

// Disabled when there's nothing more to load
<LoadMore disabled>No more results</LoadMore>
```

<Callout title="Notes">
- LoadMore is a `<button>`: style is a centered `w-fit` secondary Button (`background-100` fill, `gray-alpha-400` border) that hugs its label. It stays focusable during the loading state so keyboard users don't lose their place.
- Combine `noGap` + `noBorderRadius` to attach LoadMore to the bottom edge of a Card that uses `borderBetween`: the top border seams with the last row and the bottom stays flush with the card's rounded corner.
- Prefer `<LoadMore disabled>No more results</LoadMore>` over hiding the button once the list is exhausted. The disabled state confirms "you're at the end" instead of leaving the user wondering if more will arrive.
- For sequential navigation between separate pages (docs articles, wizard steps), use `Pagination` instead. LoadMore is for revealing more rows of the same data.
</Callout>
