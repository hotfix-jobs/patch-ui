# Radio

Single-select from a small set of mutually exclusive options. Pass the label as children and the whole row becomes the click target. For lists longer than ~6 options, reach for `Select` or `Combobox`.

## Usage

```tsx
import { Radio, RadioGroup } from "@patchui/react";

// Labeled radios (children pattern)
<RadioGroup defaultValue="weekly">
  <Radio value="daily">Daily</Radio>
  <Radio value="weekly">Weekly</Radio>
  <Radio value="monthly">Monthly</Radio>
</RadioGroup>

// Controlled
const [plan, setPlan] = useState("standard");
<RadioGroup value={plan} onValueChange={(v) => setPlan(v as string)}>
  <Radio value="starter">Starter</Radio>
  <Radio value="standard">Standard</Radio>
  <Radio value="pro">Pro</Radio>
</RadioGroup>

// Disabled (whole group)
<RadioGroup defaultValue="yes" disabled>
  <Radio value="yes">Yes</Radio>
  <Radio value="no">No</Radio>
</RadioGroup>

// Bare radios for custom layout (pass aria-label)
<Radio value="a" aria-label="A" />
```

<Callout title="Notes">
- Reach for Radio when the user must pick exactly one from a small set — 2 to 6 options where seeing every option matters (deploy region, plan tier, billing cycle). Past 6 options, use `Select`.
- Pre-select the safest default so the field reads as configured. Skip the default only when the choice has real consequences and you want a deliberate pick.
- `required` and `disabled` live on the `RadioGroup`, not individual radios. Required-on-a-single-radio is meaningless.
- Arrow keys move selection within the group. Tab moves to the next field, not the next radio.
- Bare radios (no children) work for custom layouts. Always pass `aria-label` — a nameless radio is broken for screen readers.
</Callout>
