# Field

A compound component for form fields that integrate with `<Form>` validation. Wraps Base UI's Field primitive with our styling.

**When to use Field vs `<Input label="…" error="…" />`**

- **Reach for `Input.label` / `Input.error`** when it's a single input, standalone or inside a hand-rolled form. It's one prop instead of five elements.
- **Reach for Field** when the form uses `<Form>` for native validation on submit — Field automatically wires validity state, and `FieldError` shows / hides based on that state without you managing `useState` per input.
- **Also Field** when a single label covers a group of controls (a RadioGroup, a Checkbox multi-select) — the label describes the group, not any single control.

For structured settings-page cards (title + subtitle + footer with status and actions), reach for `Section` instead — it's a different primitive for that specific layout.

## Usage

```tsx
import {
  Field,
  FieldLabel,
  FieldDescription,
  FieldError,
  FieldControl,
  Input,
} from "@patchui/react";

// Basic field with Input
<Field>
  <FieldLabel>Username</FieldLabel>
  <FieldDescription>Choose a unique username.</FieldDescription>
  <Input placeholder="e.g. jane_doe" />
</Field>

// Required field — renders a small red * after the label
<Field name="email">
  <FieldLabel required>Email</FieldLabel>
  <FieldControl type="email" required />
  <FieldError />
</Field>

// Optional field — renders "(optional)" after the label
<Field name="phone">
  <FieldLabel optional>Phone</FieldLabel>
  <Input type="tel" />
</Field>

// Error state (controlled externally)
<Field invalid>
  <FieldLabel>Password</FieldLabel>
  <Input type="password" />
  <FieldError>Password must be at least 8 characters.</FieldError>
</Field>
```

<Callout title="Notes">
- `FieldLabel` and `FieldDescription` wire `aria-labelledby` and `aria-describedby` on the control automatically — no manual `htmlFor` / `id` matching needed.
- `FieldControl` can be `<Input>`, `<Checkbox>`, `<Select>`, or any Base UI control. Field integrates with all of them via the shared primitive.
- `FieldError` prints a browser default validation message when it has no children. Pass children to override with your own copy.
- `FieldValidity` is a render prop for accessing the whole validity state (custom UI, conditional descriptions, etc.).
- `FieldItem` groups checkbox / radio rows inside a Field so the group label ties to every child.
- `validationMode` on Field wins over `<Form>`'s mode: `"onSubmit"` (default), `"onBlur"`, or `"onChange"`. Use `validationDebounceTime` to throttle onChange validation.
- The required `*` and optional `(optional)` label markers are `aria-hidden`. The `required` attribute on the underlying input is what actually enforces the constraint.
- Error message animates in with a small fade + 2px slide; unmount is instant so users see errors dismiss the moment they fix the field. Reduced-motion respected.
</Callout>
