# Toast

Stackable notification toasts built on [Sonner](https://sonner.emilkowal.ski/) with a PatchUI-styled wrapper. Supports types (success, error, warning, info, loading), descriptions, action buttons, swipe-to-dismiss, and configurable positioning. Trigger via the `toast()` function from anywhere in your app.

## Usage

```tsx
import { Toaster, toast } from "@patchui/react";

// 1. Mount Toaster once near the root of your app
function App() {
  return (
    <>
      <Toaster position="bottom-right" />
      <MyContent />
    </>
  );
}

// 2. Trigger from anywhere
toast("Hello world");
toast.success("Saved!");
toast.error("Something went wrong");
toast.warning("Check your input");
toast.info("New version available");
toast.loading("Uploading...");

// With description
toast.success("Scheduled: Catch up", {
  description: "Friday at 5:57 PM",
});

// With action button
toast.success("Message sent", {
  action: {
    label: "Undo",
    onClick: () => undoSend(),
  },
});

// Programmatic dismissal
const id = toast.loading("Uploading...");
toast.dismiss(id);
```

## PatchUI styling

PatchUI's `Toaster` is a pre-configured Sonner wrapper. Compared to Sonner defaults:

- **6px radius**, **0.5px hairline border**, **`--shadow-menu`** drop shadow
- **No background flood** for variants — instead, a **2px colored left-edge rail** in the variant color (success / warning / danger / info) keeps the surface neutral and the message restrained
- **Compact padding** matching Menu / Dialog density
- **Type scale**: title is `--text-patch-control` semibold, description is `--text-patch-mini` in `--patch-text-secondary`
- **Action button** styled like `Button variant="outline" size="sm"`
- **Close button** styled like Dialog's `×`

`Toaster` accepts all other props from Sonner's `Toaster` component (for example `expand`, `visibleToasts`, `offset`, `gap`, `richColors`). PatchUI sets `richColors: false` by default to preserve the neutral surface; setting `richColors: true` floods the toast with variant colors instead.

<Callout title="Notes">
- **Mount Toaster once** at the root of your app or layout. Multiple Toasters render their own queues.
- **Imperative API:** `toast()` and its variants (`toast.success`, `toast.error`, etc.) work from anywhere — no provider context needed (Sonner uses an internal queue).
- **Stack behavior:** Multiple toasts stack with a peek animation. Hover over the stack to expand.
- **Swipe to dismiss:** Touch and mouse drag dismissals are built in.
- **Returns an id:** Every `toast()` call returns an id you can pass to `toast.dismiss(id)` for programmatic dismissal.
- **Promise helper:** Use `toast.promise(myPromise, { loading, success, error })` to wire a toast to an async operation.
</Callout>
