# Toast

Stackable notification toasts built on Base UI Toast. Supports types (success, error, warning, info, loading), descriptions, action buttons, swipe-to-dismiss, and configurable positioning. Trigger via the `toast()` function from anywhere in your app: it forwards to a global `createToastManager()` singleton, no provider context required at the call site.

## Usage

```tsx
import { Toaster, toast } from "@patchui/react";

// 1. Mount Toaster once near the root of your app
function App() {
  return (
    <>
      <Toaster position="bottom-right" />
      <MyContent />
    </>
  );
}

// 2. Trigger from anywhere
toast("Hello world");
toast.success("Saved!");
toast.error("Something went wrong");
toast.warning("Check your input");
toast.info("New version available");
toast.loading("Uploading...");

// With description
toast.success("Scheduled: Catch up", {
  description: "Friday at 5:57 PM",
});

// With action button
toast.success("Message sent", {
  action: {
    label: "Undo",
    onClick: () => undoSend(),
  },
});

// Programmatic dismissal
const id = toast.loading("Uploading...");
toast.dismiss(id);
```

## Chrome

- **radius-12** rounded corners, **1px** hairline border, `shadow-modal` drop shadow: matches Modal / Sheet / Menu chrome.
- **Semantic icon color** signals the variant: `--success` for success, `--error` for error, `--warning` for warning. No background flood on the toast itself.
- **Compact padding** matching Menu density.
- **Typography**: title is `text-button-14`, description is `text-caption-12 text-ink-muted`.
- **Action button** renders as a bordered secondary button inside the toast.

<Callout title="Notes">
- **Mount Toaster once** at the root of your app or layout. Multiple Toasters share the same global queue.
- **Imperative API:** `toast()` and its variants (`toast.success`, `toast.error`, etc.) forward to a global `createToastManager()`. No provider context needed at the call site.
- **Stack behavior:** Toasts stack with a peek animation. Hover expands the stack; leaving it collapses again.
- **Swipe to dismiss:** Base UI ships native swipe-to-dismiss on touch and mouse drag.
- **Priority + ARIA:** `warning` and `error` announce `assertive`; everything else announces `polite`.
- **Returns an id:** Every `toast()` call returns an id you can pass to `toast.dismiss(id)` for programmatic dismissal. `toast.dismiss()` with no argument clears all.
</Callout>
