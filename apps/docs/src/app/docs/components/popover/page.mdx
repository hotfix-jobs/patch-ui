# Popover

A floating panel anchored to a trigger. Use for arbitrary content: notification lists, filter panels, mini-forms, and row action menus. When you want a single list of clickable actions, reach for [Menu](/docs/components/menu); when you want a hover chip for one line of text, reach for [Tooltip](/docs/components/tooltip); when you need a blocking dialog, reach for [Modal](/docs/components/modal).

## Usage

```tsx
import { Popover, PopoverTrigger, PopoverContent, PopoverClose } from "@patchui/react";
import { Button } from "@patchui/react";

<Popover>
  <PopoverTrigger render={<Button variant="secondary">Open</Button>} />
  <PopoverContent>
    <p className="text-body-14 text-ink">Anything you want, anchored to the trigger.</p>
    <PopoverClose render={<Button size="sm">Done</Button>} />
  </PopoverContent>
</Popover>
```

### `<PopoverClose>`

Wraps a trigger (usually a `Button`) that dismisses the popover on click. Compose via `render`.

<Callout title="Notes">
- The default `align="center"` is right for a triggerbelow-header. For an icon button in the top-right of a header (bell icon), pass `align="end"` so the popover doesn't overflow.
- Set width on `PopoverContent` via className (`w-72`, `w-[360px]`). Default `min-w-[220px]` keeps small popovers legible.
- The popover content region has no built-in padding. Pass `p-4` (for a form) or `p-0` (for a list that carries its own row padding).
- Popovers don't trap focus by default. Set `modal` on `<Popover>` if you want the surrounding page inert while the popover is open.
</Callout>
