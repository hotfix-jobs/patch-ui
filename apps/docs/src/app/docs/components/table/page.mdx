# Table

## Usage

```tsx
import {
  Table,
  TableHeader,
  TableBody,
  TableFooter,
  TableRow,
  TableHead,
  TableCell,
  TableCaption,
  TableEmpty,
} from "@patchui/react";

// Flat (default) â€” transparent frame, hairline dividers, no outer box
<Table>
  <TableHeader>
    <TableRow>
      <TableHead>Plan</TableHead>
      <TableHead align="right">Price</TableHead>
    </TableRow>
  </TableHeader>
  <TableBody>
    <TableRow>
      <TableCell>Team</TableCell>
      <TableCell align="right">$49</TableCell>
    </TableRow>
  </TableBody>
</Table>

// Elevated â€” bg-layer-1 + hairline-soft + radius-8 (matches Card)
<Table variant="elevated">...</Table>

// Row-state opinions on the root
<Table striped>...</Table>
<Table interactive>...</Table>
<Table bordered>...</Table>

// Density
<Table size="sm">...</Table>

// Selected row
<TableRow selected onClick={...}>...</TableRow>

// Sortable column head
<TableHead sortable direction="asc" onSort={...}>Name</TableHead>

// Sticky header inside a fixed-height scroll container
<div className="h-64 overflow-y-auto">
  <Table scrollable={false} stickyHeader>...</Table>
</div>

// Empty state
<TableBody>
  <TableEmpty colSpan={4}>No results.</TableEmpty>
</TableBody>
```

<Callout title="Notes">
- **State opinions live on the root.** `striped` / `interactive` / `bordered` moved from `TableBody` to `Table` this pass â€” one place to set them, propagated via context. `TableBody` is now a semantic passthrough.
- **Variant naming matches Card / Section.** `flat` vs `elevated` are the same visual axis across every surface primitive, so an elevated Table sits next to an elevated Card without any tuning.
- **Composition:** Every sub-component maps to a native table element, so standard HTML attributes pass through.
- **Numerals:** Use `align="right"` on numeric `TableCell`s for right-align + tabular figures.
- **Sortable heads:** Wire your own sort state. The head cycles the icon and sets `aria-sort` for you; the cycle logic is left to the consumer (three-state or two-state).
- **Selected rows:** Combine with `interactive` on the root for the row-clickable pattern shown in the demo.
- **Sticky header:** Requires a fixed-height scroll container. Pass `scrollable={false}` so the internal horizontal-scroll wrapper doesn't fight your vertical-scroll parent.
- **Empty state:** Wrap `<EmptyState>` inside `<TableEmpty>` for the fullest treatment (icon + title + description + action). For a one-line placeholder, pass a string directly (`<TableEmpty colSpan={4}>No results.</TableEmpty>`).
</Callout>
