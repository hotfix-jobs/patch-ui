# Label

A styled `<label>` at `text-label-14 gray-1000`. Use it standalone when you want the label positioned separately from the control, or wrap it around a `Switch` / `Checkbox` / `Radio` for click-to-toggle behavior.

For a single input where the label sits directly above, `<Input label="…" id="…" />` is the more compact path — it wires up `htmlFor` and `aria-describedby` for you.

## Usage

```tsx
import { Label, Input, Switch, Checkbox } from "@patchui/react";

// Standalone with an input
<Label htmlFor="email">Email</Label>
<Input id="email" type="email" />

// Wrapping a Switch or Checkbox — the whole label becomes clickable
<Label>
  <Switch />
  <span>Receive weekly digest</span>
</Label>

// As a different element (polymorphic render prop)
<Label render={<span />}>Read-only label</Label>
```

<Callout title="Notes">
- `<Label>` is a plain `<label>` under the hood. Every native attribute (`htmlFor`, `form`, etc.) passes through.
- For the common "label above input" pattern, `<Input label="Email" id="email" />` is one prop instead of two elements. Reach for standalone `<Label>` when the layout is custom or the control isn't an Input.
- Wrapping the label around a `Switch`, `Checkbox`, or `Radio` makes the whole label click to toggle — no `htmlFor` needed. Add `cursor-pointer` so the cursor confirms it.
- Rendered as `<span>` via `render={<span />}` when you want the visual weight without the form semantics.
</Callout>
