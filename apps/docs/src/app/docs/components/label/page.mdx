# Label

A styled `<label>` at `text-small text-ink`. Use it standalone when you want the label positioned separately from the control, or wrap it around a `Switch` / `Checkbox` / `Radio` for click-to-toggle behavior.

For a complete form-field stack, use `Field` and `FieldLabel`. Reach for standalone Label when the layout is custom or the control is outside Field.

## Usage

```tsx
import { Label, Input, Switch, Checkbox } from "@patchui/react";

// Standalone with an input
<Label htmlFor="email">Email</Label>
<Input id="email" type="email" />

// Wrapping a Switch or Checkbox — the whole label becomes clickable
<Label>
  <Switch />
  <span>Receive weekly digest</span>
</Label>

// As a different element (polymorphic render prop)
<Label render={<span />}>Read-only label</Label>
```

<Callout title="Notes">
- `<Label>` is a plain `<label>` under the hood. Every native attribute (`htmlFor`, `form`, etc.) passes through.
- Use `FieldLabel` for labels inside a Field. Reach for standalone `<Label>` when the layout is custom or the control is outside Field.
- Wrapping the label around a `Switch`, `Checkbox`, or `Radio` makes the whole label click to toggle; no `htmlFor` is needed.
- Rendered as `<span>` via `render={<span />}` when you want the visual weight without the form semantics.
</Callout>
