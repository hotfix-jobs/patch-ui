## Usage

<Usage code={usageCode} />

Textarea grows through CSS `field-sizing: content`. The `rows` attribute establishes a minimum height.

## Examples

### Size, rows, and states

Use `rows` to establish minimum space. Read-only content remains selectable, while disabled content is unavailable.

<ComponentPreview label="Size, rows, and states" sourcePath="src/app/docs/components/textarea/textarea-states-demo.tsx"><TextareaStatesDemo /></ComponentPreview>

## API reference

Textarea accepts native textarea attributes. These props select Patch UI presentation.

<PropsTable props={[
  { name: "size", type: "\"sm\" | \"md\" | \"lg\"", default: "\"md\"", description: "Sets text size and padding." },
  { name: "variant", type: "\"default\" | \"unstyled\"", default: "\"default\"", description: "Removes control chrome when another component owns the surface." },
  { name: "rows", type: "number", description: "Sets the minimum visible row count while preserving automatic growth." },
]} />

## Accessibility

- Associate every Textarea with a visible label.
- Connect helper text and errors with Field or `aria-describedby`.
- Read-only values remain selectable and copyable; disabled values do not.
