# Textarea

A multi-line text input that auto-grows with content by default. Three sizes, inline `label` and `error` message, read-only support. Uses the same `background-100` fill and translucent border stepping as `Input`.

## Usage

```tsx
import { Textarea } from "@patchui/react";

// Basic
<Textarea placeholder="Write something…" />

// Sizes
<Textarea size="sm" />   // label-12 text
<Textarea size="md" />   // copy-14 text (default)
<Textarea size="lg" />   // copy-16 text

// Fixed minimum rows
<Textarea rows={6} placeholder="Tall" />

// Inline label + error
<Textarea id="desc" label="Description" placeholder="Describe this project" />
<Textarea id="notes" label="Release Notes" error="Release notes are required." />

// Read-only
<Textarea defaultValue="Locked content" readOnly />
```

<Callout title="Notes">
- Textarea handles anything with line breaks. Anything single-value is an Input.
- Start with enough `rows` that the field feels finished at load. Growing the field beyond that is fine as long as the CTA doesn't disappear from view.
- Auto-growing is the default via `field-sizing: content`. Set `rows` when you want a specific minimum height and no more.
- `error` accepts a boolean or a string. A string renders below the field with `role="alert"`. Prefer strings — the message names what's wrong.
- Read-only is a native attribute; users can still select and copy.
- For descriptive helper text unrelated to validation, place it on a sibling element with `aria-describedby`.
</Callout>
