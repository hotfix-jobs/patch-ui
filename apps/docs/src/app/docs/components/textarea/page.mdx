# Textarea

A multi-line text input aligned with Geist. Auto-grows with content by default. Three sizes, inline `label` and `error` message, read-only support. Uses the same `background-100` fill and translucent border stepping as `Input`.

## Usage

```tsx
import { Textarea } from "@patchui/react";

// Basic
<Textarea placeholder="Write somethingâ€¦" />

// Sizes
<Textarea size="sm" />   // label-12 text
<Textarea size="md" />   // copy-14 text (default)
<Textarea size="lg" />   // copy-16 text

// Fixed minimum rows
<Textarea rows={6} placeholder="Tall" />

// Inline label + error
<Textarea id="desc" label="Description" placeholder="Describe this project" />
<Textarea id="notes" label="Release Notes" error="Release notes are required." />

// Read-only
<Textarea defaultValue="Locked content" readOnly />
```

<Callout title="Best practices">
- **Pick Textarea** when content can wrap to multiple lines (commit messages, descriptions, release notes). Use `Input` for a single value.
- **Set a generous default `rows`** and grow only when the surface has vertical room â€” don't push primary actions below the fold.
- **Validate on blur** and pass a string to `error` to render the inline message. The message replaces helper text on failure.
- **Trim** leading/trailing whitespace on submit so empty newlines don't pass a `required` check.
- **Labels** are Title Case nouns: `Description`, `Release Notes`.
- **Placeholders** show an example value, never instructions like `Enter a description`.
- **Validation messages** name the field and constraint, end in a period: `Description is required.`, `Release notes can't exceed 500 characters.`
</Callout>
