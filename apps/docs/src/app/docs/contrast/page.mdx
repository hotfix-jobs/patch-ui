# Contrast Contract

Patch UI assigns a WCAG target to each readable token and filled-label pair. `npm run check:contrast -w packages/react` evaluates those pairs in light and dark mode, and CI fails when a token change breaks the contract. The matrix below reads the active CSS variables so it updates when the page theme changes.

- **`ink`** on base, layers, and fills: AAA body text (7:1)
- **`ink-muted`** on base, `layer-1`, and `fill-1`: AA secondary text (4.5:1)
- **`ink-subtle`** on base: AA-large and placeholder treatment (3:1)
- **`ink-tertiary`** on base: AA-large and disabled hint treatment (3:1)
- **`on-primary`** on **`primary`**: AAA primary Button label (7:1)
- **Status foregrounds** on their error, warning, and success fills: AA (4.5:1)

The matrix uses check and X icons for pass or fail. Ratio labels use compact metadata geometry rather than status-colored pills.

## Choosing a text tier

- Use **`ink`** for headings, body text, and anything users must read precisely.
- Use **`ink-muted`** for secondary labels, metadata, and the default icon tone.
- Use **`ink-subtle`** for placeholders or missing-value copy on base. Use `ink-muted` on lifted or filled surfaces.
- Use **`ink-tertiary`** for disabled controls, footnotes, and timestamps, not required reading.

## What the check covers

The script parses `packages/react/src/theme/tokens.css` and checks the declared light and dark pairs. It covers the shipped token contract, not arbitrary opacity, gradients, images, or downstream CSS overrides.

If a consumer changes `--primary`, `--on-primary`, `--ink*`, `--base`, `--layer-*`, or `--fill-*`, that consumer should run an equivalent contrast check against its final values before shipping.
