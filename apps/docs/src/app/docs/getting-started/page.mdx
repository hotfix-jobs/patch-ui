# Getting Started

Patch UI is distributed as **copy-in components**. You add a component and its source lands in your own repo, fully yours to edit. There is no Patch UI npm package to install, no version to bump, and no CSS purge config to maintain.

## Prerequisites

- A React 19 project with **Tailwind CSS v4** (Next.js, Vite, Remix: any modern React setup).
- The `shadcn` CLI (run via `npx`, no install needed). It is a generic registry client that will copy Patch UI's source into your repo.

## 1. Register the Patch UI namespace

If you do not have a `components.json` yet, run `npx shadcn@latest init` once to create one. Then add the Patch UI registry under `registries`:

```json
// components.json
{
  "registries": {
    "@patchui": "https://ui.hotfix.jobs/r/{name}.json"
  }
}
```

## 2. Add a component

```bash
npx shadcn@latest add @patchui/button
```

Running `add` for the first time does four things:

1. Copies the component source into your components directory (e.g. `components/ui/button.tsx`).
2. Pulls in anything the component depends on: the `cn()` helper, the shared recipes (`focusRing`, `controlSize`, `colorTransition`), and the design tokens.
3. Installs the required npm packages (`@base-ui/react`, `class-variance-authority`, `clsx`, `tailwind-merge`, and `motion` for components with animations).
4. Copies the design tokens to `styles/patch-ui-tokens.css`.

## 3. Wire up the tokens

Import the copied tokens through your Tailwind CSS entry file (not a JS import, the `@theme` block needs Tailwind's pipeline):

```css
/* app/globals.css */
@import "tailwindcss";
@import "./styles/patch-ui-tokens.css";
```

This registers the tokens as utilities (`bg-background-100`, `text-gray-1000`, `text-copy-14`, `rounded-[var(--radius-6)]`, etc.) and drives light and dark mode simultaneously.

### Light and dark mode

Tokens cover both modes. Add the `dark` class to `<html>` to switch. No JavaScript theme engine required:

```html
<html class="dark">
```

For a UI toggle, install the `theme-toggle` component (`npx shadcn add @patchui/theme-toggle`). It handles system preference detection and persistence for you.

## 4. Use it

```tsx
import { Button } from "@/components/ui/button";

export default function App() {
  return (
    <div className="flex gap-3">
      <Button variant="primary">Save</Button>
      <Button variant="secondary">Cancel</Button>
      <Button variant="error">Delete</Button>
    </div>
  );
}
```

Button ships five variants (`primary`, `secondary`, `tertiary`, `warning`, `error`), three sizes (`sm`, `md`, `lg`), four shapes (`square`, `pill`, `circle`, `rounded`), icon slots on either side, and a loading state. See the [Button docs](/docs/components/button) for the full API.

## What you own

- **The source code.** Copied components live in your repo and are yours to edit. Rename props, change variants, delete what you do not use. Nothing is off-limits.
- **The tokens.** `styles/patch-ui-tokens.css` is copied once. Change a color there, and every component picks it up.
- **The updates.** To pull upstream changes, re-run `npx shadcn add @patchui/<name>`. Review the diff before accepting. If you have customized a component, decide what to keep.
