# Dropzone

Drag-and-drop file upload with a click-to-browse fallback. Renders a preview row per file with a remove × button. Pure UI primitive: consumers wire `onValueChange` to their upload pipeline.

Use for any file-upload UX: avatars, attachments, document uploads, data imports, or batch images.

## Usage

```tsx
import { Dropzone } from "@patchui/react";

const [files, setFiles] = useState<File[]>([]);

<Dropzone
  value={files}
  onValueChange={setFiles}
  accept="image/*,application/pdf"
  maxSize={5 * 1024 * 1024}  // 5 MB per file
  maxFiles={10}
  title="Drop files here or click to browse"
  description="Up to 5 MB, images or PDFs"
/>

// Single file (no list)
<Dropzone multiple={false} showFileList={false} />

// Custom validation
<Dropzone
  validate={(file, current) => {
    if (file.size > 2 * 1024 * 1024) return "File too large";
    if (current.some((f) => f.name === file.name)) return "Duplicate";
    return true;
  }}
/>

// Custom file row
<Dropzone
  renderFile={(file, i, onRemove) => (
    <div className="my-custom-row">
      <img src={URL.createObjectURL(file)} />
      <button onClick={onRemove}>×</button>
    </div>
  )}
/>
```

<Callout title="Notes">
- The drop area is a `<button>`: Enter / Space open the native file picker. The underlying `<input type="file">` is `sr-only`.
- While files are being dragged over, `data-drag-over` is set on the button. Default styling darkens the border to `--primary` and fills the bg with `--fill-1` so the surface reads as "yes, drop here."
- Dropzone is UI-only. It collects files into `value` (controlled) or internal state (uncontrolled). Upload logic lives in your `onValueChange` handler: call `fetch`, your storage SDK, whatever your pipeline is.
- Set `progress` (`number | null`) to shift the whole component into an uploading state: the file rows swap their size text for a `<Progress>` bar, and interaction is disabled until you clear it.
- Pass a `renderFile` if you need a fully custom row (preview thumbnail, per-file cancel, retry button). Default row shows a file icon, name, size, and an × to remove.
</Callout>
