# AppHeader

A focused site-header block for brand, primary navigation, and right-aligned actions. Below the `md` viewport breakpoint it automatically collapses navigation into a morphing hamburger trigger. Search, filter, and other route-owned tools remain siblings so applications can control their sticky and scroll behavior.

## Usage

```tsx
import {
  AppHeader,
  AppHeaderBrand,
  AppHeaderMobileTop,
  AppHeaderNav,
  AppHeaderNavItem,
  AppHeaderNavSection,
  AppHeaderRight,
} from "@patchui/react/blocks/app-header";
import { Button } from "@patchui/react";

<AppHeader>
  <AppHeaderBrand render={<a href="/" />}>Patch</AppHeaderBrand>

  <AppHeaderNav>
    <AppHeaderNavItem href="/product">Product</AppHeaderNavItem>
    <AppHeaderNavItem href="/pricing">Pricing</AppHeaderNavItem>

    {/* Sections render as titled groups inside the mobile panel.
        On desktop the items still show inline (title is dropped). */}
    <AppHeaderNavSection title="Resources">
      <AppHeaderNavItem href="/blog">Blog</AppHeaderNavItem>
      <AppHeaderNavItem href="/docs">Docs</AppHeaderNavItem>
    </AppHeaderNavSection>
  </AppHeaderNav>

  {/* Pinned into the mobile top bar just before the hamburger. Hidden
      on desktop and never rendered in the mobile panel footer — for
      one-tap actions (notification bell, avatar) that stay reachable
      without opening the panel. */}
  <AppHeaderMobileTop>
    <NotificationBell />
  </AppHeaderMobileTop>

  <AppHeaderRight>
    <Button variant="tertiary" size="sm">Sign in</Button>
    <Button variant="primary" size="sm">Get started</Button>
  </AppHeaderRight>
</AppHeader>

// Applications own compound sticky or collapsing chrome.
<motion.div animate={{ y: collapsed ? "-100%" : 0 }}>
  <AppHeader bordered={scrolled} />
  <SearchSuggestions />
  <FilterToolbar />
</motion.div>
```

## Mobile behavior

Below `md`:
- `AppHeaderNav` is hidden inline; its items render inside the panel instead.
- `AppHeaderRight` is hidden inline; its items render in the panel's sticky footer instead. Reach for that for a Sign in / primary CTA that stays parked at the bottom of the drawer.
- `AppHeaderMobileTop` renders inline just before the hamburger — visible on mobile top bar, never rendered in the panel footer. Use for one-tap actions that shouldn't require opening the panel (notification bell, avatar).
- A **morphing hamburger trigger** replaces the right cluster in the top row. It only renders when `AppHeaderNav` has children.
- Tapping the trigger drops the panel down — the hamburger morphs to X for the close affordance, and the panel occupies `top: var(--header-height, 64px)` to the bottom of the viewport. Set `--header-height` on `:root` when your header height differs from the 64px default (a `ResizeObserver` on the header element is the usual wiring).

## Styling notes

- **Desktop nav items** use radius 8 and the shared `layer-hover` interaction background. The active item keeps full ink and medium weight.
- **Borders:** `bordered` controls one structural bottom hairline. The dashboard demo shows it on; the marketing demo opts out.
- **Route tools:** place SearchSuggestions, FilterToolbar, and route actions outside AppHeader. Wrap the complete stack in application-owned sticky or collapsing chrome when those rows need to move together.
- **Header actions**: use the standard radius-8 Button shape so marketing and product controls share one vocabulary.
- **Sticky headers**: pass `sticky` to pin the header. The panel renders below at `z-40` — the sticky header (`z-50`) stays above it so brand and the hamburger-to-X trigger remain visible.
- **Escape hatch**: if you need a custom mobile pattern (drawer instead of full-viewport panel), skip this block and compose the primitives (`Sheet` + your own trigger).

<Callout title="Notes">
- **Composition:** Extract logic runs on `AppHeader`'s children. Put `AppHeaderBrand`, `AppHeaderNav`, `AppHeaderMobileTop`, and `AppHeaderRight` as direct children (order-independent) and the block wires desktop + mobile automatically.
- **Same tree, two layouts:** The nav children you write once serve both the desktop row and the mobile panel. No duplication.
- **Routing:** Use the `render` prop on `AppHeaderBrand` and `AppHeaderNavItem` to integrate your router's link component.
- **Block, not primitive:** For full control (custom mobile pattern, custom collapsing rules, drawers), compose your own header from the underlying primitives.
</Callout>
