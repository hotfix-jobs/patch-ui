# FilterToolbar

A responsive filter rail composed from Patch UI controls. It standardizes overflow, wrapping, clear-all placement, result metadata, pinned actions, and restrained motion while consumers own filter state and popup content.

## Usage

```tsx
import {
  FilterToolbar,
  FilterToolbarPicker,
  FilterToolbarTrigger,
} from "@patchui/react/blocks/filter-toolbar";

<FilterToolbar
  activeCount={activeCount}
  onClearAll={clearFilters}
  count="128 results"
  trailing={<SortMenu />}
>
  <Menu>
    <MenuTrigger
      render={<FilterToolbarTrigger label="Status" value="Active" />}
    />
    <MenuPopup>...</MenuPopup>
  </Menu>
  <FilterToolbarPicker
    label="Category"
    selected={categories}
    onSelectionChange={setCategories}
    query={query}
    onQueryChange={setQuery}
    sections={categorySections}
  />
  <CategoryFilter />
  <OwnerFilter />
</FilterToolbar>
```

<Callout title="Notes">
- FilterToolbar does not know filter values, options, URLs, or data-fetching behavior. Pass Button, Toggle, Menu, and Popover compositions as children.
- The toolbar is transparent. Each filter control owns its neutral fill and interaction states.
- FilterToolbarTrigger keeps the category label visible, adds a concise active value, and uses the shared selected fill. Clear one filter from its popup; use Clear all for the complete toolbar.
- Active triggers keep the secondary button's `fill-2` hover background at rest, on focus, and while open.
- Do not add placeholder options such as "Any" or "All". Use a distinct Clear filter action when the filter is active.
- FilterToolbarPicker is the searchable multi-select pattern for high-cardinality filters. Selection stays open, search resets on close, and the consumer owns local or async filtering.
- The clear action stays in the filter sequence so trailing actions remain stable on narrow screens.
- `scroll` is the default for headers and mobile layouts. Use `wrap` on spacious directory or management pages.
- Sticky positioning and page-width padding belong to the containing application shell.
</Callout>
